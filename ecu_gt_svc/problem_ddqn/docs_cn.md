# DDQN — 双重深度 Q 网络服务部署

## 概述

本模块训练**双重深度 Q 网络（DDQN）**智能体解决服务到 ECU 的部署任务。  
与基于 PPO 的方法（问题 3–5）不同，DDQN 采用**离策略、经验回放**训练循环，并使用两个 Q 网络以减少高估偏差，通过**奖励整形 + 硬终止**来执行约束——不使用动作掩码。

---

## 问题定义

| 符号  | 含义                   |
| ----- | ---------------------- |
| $N$   | ECU 数量               |
| $M$   | 服务数量（$M \leq N$） |
| $e_j$ | ECU $j$ 的 VM 容量     |
| $n_i$ | 服务 $i$ 的 VM 需求    |

**目标：** $\max AR = \frac{1}{M}\sum_{i,j} x_{ij} \frac{n_i}{e_j}$

---

## MDP 建模

### 状态空间

$$s_t = \bigl[\, d_t,\; AR_t,\; r_0^t,\; \ldots,\; r_{N-1}^t \,\bigr] \in \mathbb{R}^{N+2}$$

| 索引           | 内容                   | 范围    |
| -------------- | ---------------------- | ------- |
| 0              | 当前服务需求（归一化） | $[0,1]$ |
| 1              | 累积 AR                | $[0,1]$ |
| $2 \ldots N+1$ | 各 ECU 剩余容量百分比  | $[0,1]$ |

### 动作空间

$a_t \in \{0, \ldots, N-1\}$ — 离散选择目标 ECU。

### 奖励函数（密集 + 惩罚）

```
不可行分配（容量超出或重复使用 ECU）：
    奖励 = -1.0，回合立即终止

合法分配（非最后一步）：
    奖励 = ru / M   （即时奖励，与利用率成比例）

合法分配（最后一步，所有 M 个服务已部署）：
    奖励 = ru / M   （与中间步相同）
```

零违反条件下，完整 $M$ 步回合的累积奖励等于最终 AR。

| 情形         | 奖励                          | 终止 |
| ------------ | ----------------------------- | ---- |
| 不可行 ECU   | $-1$                          | ✓    |
| 合法，中间步 | $n_{a_t} / (M \cdot e_{a_t})$ | ✗    |
| 合法，最后步 | $n_{a_t} / (M \cdot e_{a_t})$ | ✓    |

### 与 PPO 方法的关键区别

| 方面       | DQN（本模块）    | P3 PPO          | P4 PPO + 掩码 | P5 PPO + 拉格朗日 |
| ---------- | ---------------- | --------------- | ------------- | ----------------- |
| 奖励       | **密集**（每步） | 稀疏（末步 AR） | 稀疏          | 密集（拉格朗日）  |
| 约束处理   | 终止 + 惩罚      | 无              | 硬掩码        | 软惩罚            |
| 在/离策略  | **离策略**       | 在策略          | 在策略        | 在策略            |
| 回放缓冲区 | ✓                | ✗               | ✗             | ✗                 |

## 算法与超参数

**算法：** DQN（Stable-Baselines3）

| 超参数            | 值                       |
| ----------------- | ------------------------ |
| 学习率            | `1e-3`                   |
| 回放缓冲区大小    | `100,000`                |
| 开始学习步数      | `2,000`                  |
| 批次大小          | `64`                     |
| 目标网络 $\tau$   | `1.0`（硬更新）          |
| 折扣因子 $\gamma$ | `0.99`                   |
| 训练频率          | 每 `4` 步                |
| 梯度步数          | `1`                      |
| 目标网络更新间隔  | 每 `500` 步              |
| 探索比例          | `0.5`（总步数的前 50 %） |
| 最终 $\epsilon$   | `0.05`                   |
| Q 网络隐层        | `[128, 128]`             |
| 总训练步数        | `500,000`                |

探索策略：$\epsilon$ 在训练前 50 % 的步数内从 `1.0` 线性衰减至 `0.05`。

| `env.py`      | `DQNEnv` — 密集奖励，违反时终止回合             |

| `results/`    | 模型、训练日志、图表、汇总 CSV                  |

## 使用方法

```bash
# 完整流水线
python dqn/run_all.py

# 仅训练
python dqn/train.py

# 仅评估
python dqn/evaluate.py
```

## 设计说明

- **密集奖励**在每步提供即时信号，相比稀疏奖励 PPO 可加速早期学习。
- **违反时终止回合**是简单有效的硬约束机制，无需 `sb3-contrib` 或拉格朗日调参。
- 由于 DQN 使用**经验回放缓冲区**，对离散动作空间可能比在策略 PPO 更高效地利用样本。
- Q 网络使用 `[128, 128]` 的较小结构，因为 DQN 更新目标噪声更大，大型网络容易对缓冲区中的过时转移过拟合。
- 总训练预算（`500,000` 步）大于 P3/P4（`200,000` 步），以补偿训练初期缓冲区多样性不足导致的离策略样本相关性问题。
