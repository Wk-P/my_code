RL Algorithms — Core Update Mathematics (Bellman Level)

DQN / DDQN Off-policy · Value Function

Standard DQN — Bellman Optimality Equation

$$Q^*(s,a) = \mathbb{E}\bigl[r + \gamma \max_{a'} Q^*(s', a')\bigr]$$

Q-Target (using target network $\theta^-$ during training)

$$y_t = r_t + \gamma \cdot \max_{a'} Q(s_{t+1},\, a';\, \theta^-)$$

DDQN Improvement (reduces overestimation bias)

$$y_t = r_t + \gamma \cdot Q\!\Bigl(s_{t+1},\; \underbrace{\arg\max_{a'} Q(s_{t+1}, a';\, \theta)}_{\text{online network selects action}};\; \theta^-\Bigr)$$

The online network $\theta$ selects the action; the target network $\theta^-$ evaluates it — decoupling the two roles.

Parameter Update (Huber Loss)

$$\mathcal{L}(\theta) = \mathbb{E}\bigl[(y_t - Q(s_t, a_t;\, \theta))^2\bigr]$$

$$\theta \;\leftarrow\; \theta - \alpha \cdot \nabla_\theta \mathcal{L}(\theta)$$

Target Network Update (hard copy every 500 steps)

$$\theta^- \leftarrow \tau\,\theta + (1-\tau)\,\theta^- \quad (\tau = 1.0 \Rightarrow \text{full hard copy})$$

$\varepsilon$-Greedy Exploration Decay

$$\varepsilon(t) = \max\!\left(\varepsilon_{\text{final}},\;\varepsilon_{\text{init}} - \frac{(\varepsilon_{\text{init}} - \varepsilon_{\text{final}}) \cdot t}{T \times 0.1}\right)$$

Decays from 1.0 to 0.0 over the first 10% of training steps, then pure greedy thereafter.

γ = 0.99 lr = 1e-3 τ = 1.0 buffer = 100 K batch = 64 target_update = 500 steps net = [128, 128]

PPO — P3 / P4 / P6 On-policy · Policy Gradient

Step 1 — TD Error (per step)

$$\delta_t = r_t + \gamma \cdot V(s_{t+1}) - V(s_t)$$

Step 2 — GAE (Generalized Advantage Estimation)

$$\hat{A}_t = \sum_{l=0}^{\infty} (\gamma \lambda_{\text{GAE}})^l \cdot \delta_{t+l}$$

$\lambda_{\text{GAE}} \to 1$: low bias, high variance (approaches Monte Carlo);  $\lambda_{\text{GAE}} \to 0$: high bias, low variance (approaches TD(0)). This project uses $\lambda_{\text{GAE}} = 0.95$ to balance both ends.

Step 3 — Value Function Target and Loss

$$V^{\text{target}}_t = \hat{A}_t + V(s_t)$$

$$\mathcal{L}_{\text{VF}} = \mathbb{E}\bigl[(V(s_t) - V^{\text{target}}_t)^2\bigr]$$

Step 4 — PPO-Clip Policy Objective

$$r_t(\theta) = \frac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{\text{old}}}(a_t \mid s_t)}$$

$$\mathcal{L}_{\text{CLIP}} = \mathbb{E}\!\left[\min\!\left(r_t(\theta)\,\hat{A}_t,\;\operatorname{clip}(r_t(\theta),\,1-\varepsilon,\,1+\varepsilon)\,\hat{A}_t\right)\right]$$

The clip prevents excessively large policy updates in a single step. $\varepsilon = 0.2$.

Step 5 — Total Loss (joint optimisation)

$$\mathcal{L}(\theta) = -\mathcal{L}_{\text{CLIP}} + c_1 \cdot \mathcal{L}_{\text{VF}} - c_2 \cdot \mathcal{H}[\pi_\theta]$$

$\mathcal{H}[\pi_\theta]$: policy entropy bonus to prevent premature convergence. $c_1, c_2$ are SB3 default coefficients.

γ = 0.999 (P3/P4/P6) λ_GAE = 0.95 ε_clip = 0.2 lr = 3e-4 n_steps = 512 batch = 256 epochs = 10 net = [256, 256]

P5 — PPO + Lagrangian Dual Ascent Constrained MDP · Dual Method

The inner PPO update steps are identical to the above. The key difference is that the reward signal itself contains $\lambda$:

$$r_t = \underbrace{\frac{\text{req}_t}{\text{cap}_j \cdot n_{\text{active}}}}_{\text{utilisation gain}} + \underbrace{(-2.0) \cdot \mathbf{1}[\text{cap violated}]}_{\text{fixed capacity penalty}} - \underbrace{(\lambda + 0.2) \cdot c_t}_{\text{Lagrangian conflict penalty}} + \underbrace{\text{AR} \cdot \mathbf{1}[\text{done} \wedge \text{no viol}]}_{\text{terminal bonus}}$$

$c_t = 1$ if a conflict violation occurs at the current step, otherwise $c_t = 0$.

Outer Dual Variable Update (every 200 episodes)

$$\bar{v} = \frac{1}{W}\sum_{ep \in \text{window}} \text{conf\_viol\_rate}_{ep}$$

$$\lambda \;\leftarrow\; \operatorname{clip}\!\bigl(\lambda + \alpha(\bar{v} - v^*),\;0,\;\lambda_{\max}\bigr)$$

$\bar{v}$: mean conflict violation rate over the window;  $v^* = 0$ (target: zero violations);  $\alpha = 0.01$;  $\lambda_{\max} = 5.0$.

Bi-level Optimisation Structure

$$\max_\theta \min_{\lambda \geq 0}\;\mathcal{L}(\theta, \lambda) = \mathbb{E}\!\left[\sum_t r_t^{\text{util}}\right] - \lambda \cdot \mathbb{E}\!\left[\sum_t c_t\right]$$

Inner loop: PPO performs gradient ascent on $\theta$ (maximises reward for fixed $\lambda$).
Outer loop: Gradient ascent on $\lambda$ (if violation rate exceeds target, $\lambda$ increases, driving violations toward zero).

γ = 0.99 λ_GAE = 0.95 ε_clip = 0.2 lr = 3e-4 λ_init = 0.5 λ_lr = 0.01 λ_max = 5.0 window = 200 ep net pi=[256,256] vf=[512,512]

Side-by-Side Comparison

Algorithm Core Equation Type Update Target Constraint Mechanism
DQN Bellman Optimality $Q^* \leftarrow r + \gamma \max Q^*$ Large negative penalty on violation; terminal $\pm AR$
DDQN Decoupled Bellman $\theta$ selects action, $\theta^-$ evaluates Same as DQN
P3 PPO Policy Gradient + GAE + Clip $\theta \leftarrow \theta + \alpha \nabla \mathcal{L}_{\text{CLIP}}$ Violations recorded only, never blocked
P4 PPO Policy Gradient + GAE + Clip Same as P3 Hard action masking (invalid actions never sampled)
P5 PPO+Lag Policy Gradient + Dual Ascent $\theta$ maximises $\lambda$-penalised reward; $\lambda$ via dual ascent Soft constraint: $(\lambda + 0.2) \cdot c_t$
P6 PPO Policy Gradient + GAE + Clip Same as P3 Best-fit repair heuristic (post-hoc correction)

Relationship Between the Three Families

DQN: Directly fits $Q^*$ to satisfy the Bellman optimality equation → optimal policy recovered as $\pi^*(s) = \arg\max_a Q^*(s,a)$.

PPO: Does not estimate $Q^*$ directly. Uses GAE to construct the advantage $\hat{A}_t \approx Q^\pi - V^\pi$, then updates the policy with a clipped surrogate objective to bound step size.

P5: Adds a primal-dual iteration on top of PPO, relaxing the constrained MDP into an unconstrained Lagrangian dual problem solved by alternating inner (PPO) and outer ($\lambda$) updates.
