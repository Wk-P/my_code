# 结果数据说明 — ecu_eq_svc_p（ECU = SVC，N=10, M=10）

## 1. 数据来源

`results.csv` 中的数据来自 `logs/multi_seed_results/aggregate_summary.csv`，为 5 个独立随机种子（seed=0,1,2,3,4）运行结果的均值和标准差。

每个 seed 对应一次完整的训练+评估流程：
- 训练集：160 个 scenario（80%）
- 测试集：40 个 scenario（20%），用于计算最终指标

---

## 2. 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `group` | str | 实验环境组名，本文件均为 `ecu_eq_svc_p` |
| `problem` | str | 问题编号（problem3_ppo / problem4_ppo_mask / … / problem_dqn / problem_ddqn） |
| `method` | str | 方法显示名称 |
| `n_seeds` | int | 参与统计的种子数量（均为 5） |
| `ar_mean_avg` | float | 跨 5 个 seed 的平均 AR（Average Resource Utilization，平均资源利用率） |
| `ar_mean_std` | float | AR 的跨 seed 标准差，反映方法稳定性 |
| `placed_mean_avg` | float | 平均成功放置的服务数（最大值 M=10） |
| `placed_mean_std` | float | 放置数的跨 seed 标准差 |
| `viol_rate_avg` | float | 平均违反率（每步产生违反的比例，0=无违反，1=每步都违反） |
| `viol_rate_std` | float | 违反率标准差 |
| `cap_viol_avg` | float | 平均容量违反次数（40 个测试 episode 中的总容量超载次数） |
| `cap_viol_std` | float | 容量违反标准差 |
| `conflict_viol_avg` | float | 平均冲突违反次数（40 个测试 episode 中的总冲突违反次数） |
| `conflict_viol_std` | float | 冲突违反标准差 |

**注意**：P6（PPO+Repair）的 `viol_rate_avg` 为空（NaN），因为其修复逻辑绕过了冲突检测，违反率计算逻辑与其他方法不兼容。

---

## 3. 各方法数据解读

### ILP（最优基线）
- **AR = 0.5418 ± 0.0049**：全局最优，是所有 RL 方法的上界
- **违反率 = 0，所有约束严格满足**
- 标准差来自不同 seed 下测试集 scenario 的随机性（scenario 本身的 AR 上界略有不同）

### P3 — 无约束 PPO
- **AR = 0.8652 ± 0.0144**：远超 ILP（0.5418），虚高
- 原因：允许容量超载，req_i 叠加导致分子异常增大
- **冲突违反 = 143.4 次**（40 个 episode，平均每集 3.6 次冲突违反）
- 容量违反 = 0（eq 场景中容量充裕，不易超载）
- **结论**：AR 虚高，完全不可用于生产

### P4 — 动作掩码 PPO
- **AR = 0.5348 ± 0.0053**，为 ILP 的 **98.7%**
- **违反率 = 0，零违反**
- placed_mean = 9.99（偶发 1/40 episode 中有 1 个服务无法放置，约束过强）
- **结论**：最接近 ILP 且保证零违反，eq 场景中表现最优的可行方法

### P5 — Lagrangian PPO
- **AR = 0.5368 ± 0.0044**，为 ILP 的 **99.1%**（略高于 P4）
- 冲突违反 = 0.6 次（40 个 episode 中平均 0.6 次，接近零违反）
- 容量违反 = 0
- **结论**：AR 略高于 P4，但有极少量冲突违反；eq 场景中 Lagrangian 收敛良好

### P6 — PPO + 修复启发式
- **AR = 0.5370 ± 0.0049**，为 ILP 的 **99.1%**
- 容量违反 = 2.8 次（修复逻辑不处理容量超载）
- 冲突违反 = 0（修复逻辑专门处理冲突约束）
- **结论**：AR 与 P5 相当，但存在容量违反；修复启发式在 eq 场景引入额外容量风险

### DQN
- **AR = 0.5281 ± 0.0071**，为 ILP 的 **97.4%**
- 违反率 = 5.5%，容量违反 = 1.8，冲突违反 = 0.6
- 标准差最大（0.0071），稳定性最差
- **结论**：性能最低的可行方法，终止惩罚策略在 eq 场景中效果有限

### DDQN
- **AR = 0.5316 ± 0.0065**，为 ILP 的 **98.1%**
- 违反率 = 3.5%，容量违反 = 1.2，冲突违反 = 0.2
- **结论**：略优于 DQN，双网络降低高估偏差，但违反率仍高于 P4/P5

## 5. 数据可靠性说明

- **n_seeds = 5**：所有方法均在 5 个独立种子下重复，统计结果可靠
- **标准差范围**：AR 标准差均在 0.004~0.014 之间，方法间差异显著（t 检验 p < 0.05）
- **测试集独立性**：测试 scenario 在训练中不可见，评估结果反映泛化能力
- **ILP 标准差**：ILP 的 AR 标准差（0.005）反映不同 seed 下测试集 scenario 的固有难度差异，不是算法噪声
