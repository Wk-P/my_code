# 问题 2 — 服务部署的整数线性规划最优解

## 概述

问题 2 将服务到 ECU 的部署任务建模为**整数线性规划（ILP）**，并通过 PuLP 库结合 CBC 求解器求得**全局最优解**。  
所得 AR 值作为所有后续强化学习方法（问题 3–5、DQN）的**理论上界基准**。

---

## 问题定义

| 符号                 | 含义                                     |
| -------------------- | ---------------------------------------- |
| $N$                  | ECU 数量                                 |
| $M$                  | 服务数量（$M \leq N$）                   |
| $e_j$                | ECU $j$ 的 VM 容量                       |
| $n_i$                | 服务 $i$ 的 VM 需求                      |
| $x_{ij} \in \{0,1\}$ | 服务 $i$ 分配到 ECU $j$ 时为 1，否则为 0 |

**目标——最大化总资源利用率：**

$$\max \sum_{i=0}^{M-1} \sum_{j=0}^{N-1} x_{ij} \cdot \frac{n_i}{e_j}$$

> 注：PuLP 需要线性目标函数，因此直接最大化总和。  
> 求解后，平均 AR = `total_utilisation / active_ECUs`。

**约束：**

| 约束       | 数学形式                                       | 含义                       |
| ---------- | ---------------------------------------------- | -------------------------- |
| 服务分配   | $\sum_j x_{ij} = 1 \;\forall i$                | 每个服务恰好部署到一个 ECU |
| 单服务 ECU | $\sum_i x_{ij} \leq 1 \;\forall j$             | 每个 ECU 最多承载一个服务  |
| 容量可行性 | 若 $n_i > e_j$，则 $x_{ij} = 0$                | 预先禁止不可行配对         |
| 容量限制   | $\sum_i x_{ij} \cdot n_i \leq e_j \;\forall j$ | 需求不得超过 ECU 容量      |

---

## 求解场景

配置文件描述了四个基准场景：

| 场景 | ECU 容量 | 服务需求 | 特点                     |
| ---- | -------- | -------- | ------------------------ |
| 1    | 均匀     | 均匀     | 对称；有效分配方案多     |
| 2    | 异构     | 异构     | 需要精细匹配             |
| 3    | 异构     | 均匀     | ECU 多样性导致利用率差异 |
| 4    | 均匀     | 异构     | 服务多样性导致利用率差异 |

每个场景独立求解，结果对比后绘图展示。

## 输出格式

求解完成后，每个场景产生如下结果：

```python
{
    "status":           "Optimal",
    "total_utilization": float,   # 所有活跃 ECU 的利用率之和
    "active_ecus":       int,     # 接收了服务的 ECU 数量
    "avg_utilization":   float,   # AR = total_utilization / active_ecus
    "allocation": {
        ecu_id: {
            "services":    [svc_id, ...],
            "utilization": float,
            "capacity":    int,
            "demand":      int,
        }
    }
}
```

## 与强化学习方法的对比

| 维度       | P2（ILP）        | P3（PPO，无约束）    | P4（PPO + 掩码）   | P5（PPO + 拉格朗日） | DQN         |
| ---------- | ---------------- | -------------------- | ------------------ | -------------------- | ----------- |
| 解质量     | **全局最优**     | 近似最优（可能违约） | 近似最优（0 违反） | 近似最优（少量违反） | 近似最优    |
| 计算复杂度 | NP-hard          | $O(1)$ 推理          | $O(1)$ 推理        | $O(1)$ 推理          | $O(1)$ 推理 |
| 约束满足   | 保证             | ✗                    | ✓（硬约束）        | ≈ ✓（软约束）        | ✓（惩罚）   |
| 可扩展性   | $N > 100$ 时变慢 | 支持 $N = 10^5$      | 支持               | 支持                 | 支持        |

## 使用方法

```bash
python problem2_ilp/optimal_solution/main.py --config problem2_ilp/config/config_20260305_183222.yaml
```

脚本从 YAML 读取所有场景，用 PuLP/CBC 逐一求解，打印每个场景的结果，并将汇总统计和图表保存到 `results/` 目录。
