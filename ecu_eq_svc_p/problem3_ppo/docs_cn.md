# 问题 3 — 无约束执行的 PPO 服务部署

## 概述

问题 3 在服务部署任务上训练**近端策略优化（PPO）**智能体，**不执行任何约束**。  
约束违反情况仅被**记录**用于分析，不施加任何惩罚，回合始终运行完整的 $M$ 步。  
目标是衡量无约束 RL 智能体最大化 AR 的能力，并量化违反率，作为问题 4 和 5 的基准对比。

---

## 问题定义

| 符号  | 含义                   |
| ----- | ---------------------- |
| $N$   | ECU 数量               |
| $M$   | 服务数量（$M \leq N$） |
| $e_j$ | ECU $j$ 的 VM 容量     |
| $n_i$ | 服务 $i$ 的 VM 需求    |

**目标——最大化平均资源利用率（AR）：**

$$\max \; AR = \frac{1}{M} \sum_{j} \sum_{i} x_{ij} \cdot \frac{n_i}{e_j}$$

---

## MDP 建模

### 状态空间

$$s_t = \bigl[\, d_t,\; AR_t,\; r_0^t,\; \ldots,\; r_{N-1}^t \,\bigr] \in \mathbb{R}^{N+2}$$

| 索引           | 内容                                  | 范围      |
| -------------- | ------------------------------------- | --------- |
| 0              | 当前服务需求（以 $\max e_j$ 归一化）  | $[-1, 1]$ |
| 1              | 第 $t$ 步的累积 AR                    | $[0, 1]$  |
| $2 \ldots N+1$ | 各 ECU 剩余容量百分比（超载时可为负） | $[-1, 1]$ |

### 动作空间

$a_t \in \{0, \ldots, N-1\}$ — 为当前服务选择目标 ECU。

### 奖励函数（稀疏，无约束惩罚）

$$r_t = \begin{cases} AR_M & t = M-1 \text{（最后一步）} \\ 0 & \text{其他} \end{cases}$$

对容量违反或重复 ECU 分配均不施加惩罚。

### 回合结构

无论是否发生违反，每个回合均运行恰好 $M$ 步。  
不可行分配照常执行（`remaining_vms` 可变为负数）。

---

## 约束追踪（仅用于评估）

评估过程中，环境记录以下指标：

| 指标                        | 描述                            |
| --------------------------- | ------------------------------- |
| `capacity_violations`       | 需求超过 ECU 剩余容量的步骤数   |
| `single_service_violations` | 将服务分配到已使用 ECU 的步骤数 |
| `violation_rate`            | 总违反次数 / $M$                |

这些指标**不纳入奖励**，仅记录在 `info` 中。

---

## 算法与超参数

**算法：** PPO（Stable-Baselines3）

| 超参数                | 值                                         |
| --------------------- | ------------------------------------------ |
| 学习率                | `3e-4`                                     |
| 展开长度（`n_steps`） | `1024`                                     |
| 批次大小              | `128`                                      |
| 每次更新轮数          | `10`                                       |
| 折扣因子 $\gamma$     | `0.999`（较高；末步奖励需反向传播 $M$ 步） |
| GAE $\lambda$         | `0.95`                                     |
| 裁剪范围              | `0.2`                                      |
| 网络结构              | `[256, 256]`（Actor 和 Critic 共享 MLP）   |
| 总训练步数            | `200,000`                                  |

---

## 与其他方法的对比

|          | P2（ILP）      | **P3（PPO，无约束）** | P4（PPO + 掩码）   | P5（PPO + 拉格朗日） |
| -------- | -------------- | --------------------- | ------------------ | -------------------- |
| 约束执行 | 硬约束（保证） | **无**                | 硬约束（动作掩码） | 软约束（惩罚）       |
| 违反率   | 0 %            | 可能 > 0 %            | ≈ 0 %              | ≈ 0 %                |
| AR 性能  | 最优上界       | 较高（但可能违约）    | 略低               | 略低                 |
| 推理速度 | 慢（NP-hard）  | $O(1)$                | $O(1)$             | $O(1)$               |

## 文件说明

| 文件             | 描述                                                |
| ---------------- | --------------------------------------------------- |
| `env_p3.py`      | `P3Env` — 仅追踪违反，无惩罚的 Gymnasium 环境       |

| `results/`       | 已保存的模型、训练日志、图表、汇总 CSV              |

## 使用方法

```bash
# 完整流水线（推荐）
python problem3_ppo/run_all.py

# 仅训练
python problem3_ppo/train_p3.py

# 仅评估（需已有训练模型）
python problem3_ppo/evaluate_p3.py
```

---

## 预期行为

- 智能体将学会最大化 AR，可达到接近 ILP 上界的值。
- 但由于不执行任何约束，预期存在非零违反率，尤其是在需要将大需求服务匹配大容量 ECU 时。
- 违反率作为评估问题 4 和 5 约束处理策略的基准。
