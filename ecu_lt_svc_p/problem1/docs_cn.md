# 问题 1：服务部署基础强化学习环境

## 概述

问题 1 定义了服务到 ECU 部署任务的**基础 Gym 环境**。  
它是所有后续强化学习方法（问题 3–5、DQN）的基础框架。

---

## 问题定义

| 符号                 | 含义                                  |
| -------------------- | ------------------------------------- |
| $N$                  | ECU（电子控制单元）数量               |
| $M$                  | 需要部署的服务数量（$M \leq N$）      |
| $e_j$                | ECU $j$ 的 VM 容量                    |
| $n_i$                | 服务 $i$ 的 VM 需求                   |
| $x_{ij} \in \{0,1\}$ | 决策变量：服务 $i$ 是否分配到 ECU $j$ |

**目标——最大化平均资源利用率（AR）：**

$$\max \; AR = \frac{1}{M} \sum_{j=0}^{N-1} \sum_{i=0}^{M-1} x_{ij} \cdot \frac{n_i}{e_j}$$

**约束：**

- 每个服务恰好分配到一个 ECU。
- 每个 ECU 最多承载一个服务。
- 分配的服务的 VM 需求不得超过 ECU 容量。

---

## 环境设计

### 状态空间

在步骤 $t$，观测向量形状为 $(N + 2)$：

$$s_t = \bigl[\, d_t,\; AR_t,\; r_0^t,\; r_1^t,\; \ldots,\; r_{N-1}^t \,\bigr]$$

| 索引           | 内容                  | 范围    | 说明                       |
| -------------- | --------------------- | ------- | -------------------------- |
| 0              | 服务需求（归一化）    | $[0,1]$ | $n_t / \max(e_j)$          |
| 1              | 当前累积 AR           | $[0,1]$ | 已分配步骤的平均利用率     |
| $2 \ldots N+1$ | 各 ECU 剩余容量百分比 | $[0,1]$ | $\text{remaining}_j / e_j$ |

### 动作空间

$$a_t \in \{0, 1, \ldots, N-1\}$$

智能体选择将当前服务分配到哪个 ECU。

### 奖励函数

```
• 不可行分配（容量不足）：奖励 = -1.0，回合立即终止。
• 合法步骤但非最后一步：奖励 =  0.0
• 最后一步（所有 M 个服务已部署）：奖励 = 最终 AR
```

这是**稀疏 + 惩罚**设计：智能体被阻止生成不可行分配，唯一的正向信号是回合结束时的最终 AR。

### 回合结构

```
reset()
  └── 步骤 0：分配 service_0 → ECU[a_0]
       步骤 1：分配 service_1 → ECU[a_1]
       ...
       步骤 M-1：分配 service_{M-1} → ECU[a_{M-1}]  → 奖励 = AR
```

若智能体选择剩余容量不足的 ECU，回合立即以 `reward = -1` 终止。

## 设计说明

- **不可行动作触发终止**相当于隐式硬约束。智能体无需显式掩码即可学会避免容量溢出。
- 观测中包含**剩余容量追踪**，使智能体能做出类贪心决策（将大需求服务分配给大容量 ECU）。
- 所有维度的观测空间边界为 `[0.0, 1.0]`。
- 该环境作为**教学基准**使用；后续问题将放宽或修改约束处理策略。
