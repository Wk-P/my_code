# 问题 5 — 拉格朗日约束松弛的 PPO 服务部署

## 概述

问题 5 训练标准 **PPO 智能体**，结合**拉格朗日约束松弛**，将部署约束作为**软惩罚**而非硬屏蔽处理。  
训练过程中维护一个拉格朗日乘子 $\lambda$，通过**对偶上升**自适应更新，逐步加大对约束违反的惩罚，直到智能体学会避免违反。

---

## 问题定义

与问题 3 和 4 相同：

$$\max \; AR = \frac{1}{M} \sum_{i,j} x_{ij} \cdot \frac{n_i}{e_j}$$

约束：

1. 每个服务恰好分配到一个 ECU。
2. 每个 ECU 最多承载一个服务。
3. $\sum_i x_{ij} n_i \leq e_j$，对所有 $j$（容量约束）。

在问题 5 中，这些约束**不被硬屏蔽**，而是通过奖励中的拉格朗日项加以惩罚。

---

## MDP 建模

### 状态空间

$$s_t = \bigl[\, d_t,\; AR_t,\; r_0^t,\; \ldots,\; r_{N-1}^t \,\bigr] \in \mathbb{R}^{N+2}$$

| 索引           | 内容                                    | 范围      |
| -------------- | --------------------------------------- | --------- |
| 0              | 服务需求（归一化）                      | $[0,1]$   |
| 1              | 累积 AR                                 | $[0,1]$   |
| $2 \ldots N+1$ | 各 ECU 剩余容量百分比（超载时可 $< 0$） | $[-2, 1]$ |

### 动作空间

$a_t \in \{0, \ldots, N-1\}$ — 无掩码；所有 ECU 始终可选。

### 奖励函数（拉格朗日）

每步 $t$ 计算违反指示变量：

$$c_t = \begin{cases} 1 & \text{若 remaining\_vms}[a_t] < n_t \;\text{或}\; \text{ECU}\;a_t\;\text{已被分配} \\ 0 & \text{其他} \end{cases}$$

每步奖励为：

$$r_t = \frac{n_{a_t}}{M \cdot e_{a_t}} - \lambda \cdot c_t$$

其中 $\lambda \geq 0$ 为**拉格朗日乘子**（对偶变量）。

零违反时，完整 $M$ 步回合的累积奖励等于 AR。

### 回合结构

回合始终运行 $M$ 步，`remaining_vms` 可变为负数（ECU 超载）。

---

## 拉格朗日乘子更新（对偶上升）

$\lambda$ 由训练回调每 `LAMBDA_UPDATE_WINDOW` 个回合更新一次：

$$\lambda \leftarrow \text{clip}\!\bigl(\lambda + \alpha \cdot (\bar{v} - v^*),\; 0,\; \lambda_{\max}\bigr)$$

| 符号             | 含义                             | 默认值   |
| ---------------- | -------------------------------- | -------- |
| $\alpha$         | 对偶上升步长（`LAMBDA_LR`）      | `0.05`   |
| $\bar{v}$        | 近期平均每步违反率               | 在线测量 |
| $v^*$            | 目标违反率（`LAMBDA_TARGET`）    | `0.0`    |
| $\lambda_{\max}$ | $\lambda$ 最大值（`LAMBDA_MAX`） | `10.0`   |
| 窗口             | 用于估计 $\bar{v}$ 的回合数      | `30`     |

当 $\bar{v} > v^*$ 时，$\lambda$ 增大，使违反代价更高，引导策略远离不可行动作。

## 算法与超参数

**算法：** PPO（Stable-Baselines3，无掩码）

| 超参数         | 值           |
| -------------- | ------------ |
| 学习率         | `3e-4`       |
| `n_steps`      | `2048`       |
| 批次大小       | `256`        |
| 更新轮数       | `10`         |
| $\gamma$       | `0.99`       |
| GAE $\lambda$  | `0.95`       |
| 裁剪范围       | `0.2`        |
| Actor 网络     | `[128, 128]` |
| Critic 网络    | `[256, 256]` |
| 总训练步数     | `300,000`    |
| 初始 $\lambda$ | `0.0`        |

## 与其他方法的对比

|                    | P3（PPO，无约束） | P4（PPO + 掩码） | **P5（PPO + 拉格朗日）** |
| ------------------ | ----------------- | ---------------- | ------------------------ |
| 约束处理           | 无                | 硬约束（掩码）   | **软约束（惩罚）**       |
| 违反率             | > 0 %             | 0 %              | ≈ 0 %（自适应）          |
| 密集奖励           | 否（稀疏）        | 否（稀疏）       | **是**                   |
| 需要 `sb3-contrib` | 否                | 是               | 否                       |
| 需调 $\lambda$     | 否                | 否               | **是**                   |

## 设计说明

- **密集奖励**（每步 `ru/M`）比 P3/P4 的稀疏末步 AR 奖励提供更丰富的梯度信号，可加速早期学习。
- **自适应 $\lambda$** 避免了手动调整固定惩罚系数的需要；对偶上升循环会自动找到将违反率降至目标值的最小 $\lambda$。
- 与 P4 不同，智能体可以访问不可行 ECU；在宽松约束场景下这可能带来更高的 AR，但需要 $\lambda$ 足够大才能阻止违反。
- 若 $\lambda$ 增长过慢，智能体可能在惩罚显著化之前就锁定了高 AR 但高违反率的策略。
