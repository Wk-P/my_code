"""
P4 Environment — Hard capacity AND hard conflict (both masked).

N = M: each ECU hosts exactly one service on average.

Constraints:
    - Capacity violation → HARD (action masking strictly prevents selection).
    - Conflict violation → HARD (action masking strictly prevents selection).
    If no valid ECU exists, action_masks() returns all-False.

Reward (potential-based shaping):
    Per step:  Δar = ar_new - ar_prev  (incremental AR contribution)
    Terminal:  +ar_final               (completion bonus; episode return = 2 * ar_final)

Services are sorted descending by requirement at each episode reset (FFD order),
so the hardest-to-place service is always presented first.
"""

class P4Env(gym.Env):
    """
    Each episode assigns M services to N ECUs (N = M), one service per step.
    Multiple services may share ECUs.

    action_masks() returns True only for ECUs that satisfy BOTH capacity
    and conflict constraints.  If no such ECU exists, the fallback ECU
    with maximum remaining capacity is used (heavy penalty applied).

    Observation (shape: 5N+6+M):
        [0]          current service demand (normalised)
        [1]          current cumulative AR
        [2]          sum of remaining ECU capacity (normalised, clipped ≥ 0)
        [3]          sum of remaining service demand (normalised)
        [4]          fraction of ECUs with sufficient capacity for current service
        [5]          fraction of services remaining
        [6:6+N]      initial capacity fraction per ECU
        [6+N:6+2N]   remaining capacity fraction per ECU
        [6+2N:6+3N]  conflict flag per ECU (1 = placing current svc here violates a conflict set)
        [6+3N:6+4N]  ECU allowed fraction (fraction of SVCs still placeable without conflict)
        [6+4N:6+5N]  valid-action flags (1 = capacity OK AND no conflict)
        [6+5N:6+5N+M] remaining service demands (sorted descending)

    Reward:
        Δar = ar_new - ar_prev   (potential-based per-step reward)
        terminal_bonus: +ar_final (zero violations) or 0 (any violation)
    """

观测向量维度 = 5N + 6 + M。其中 N 是 ECU 数，M 是服务数。比如 N=7, M=10，则维度 = 47。

What P4Env solves: Place M services onto N ECUs (N < M), one service per timestep. Multiple services can share the same ECU.

Two hard constraints: ① Capacity — a service cannot be placed if it exceeds the ECU's remaining room; ② Conflict — some pairs/groups of services are forbidden from sharing an ECU. Both are enforced via action masking — the agent never even sees illegal actions as valid options.

Observation vector size = 5N + 6 + M. For N=7, M=10, the shape is 47.

obs_shape = 5×N + 6 + M
Action masking vs. penalty: Instead of letting the agent pick a bad ECU and then punishing it, P4 simply hides bad ECUs from the agent. The MaskablePPO algorithm from sb3-contrib zeros out the logits (log-probabilities) for masked actions before sampling, guaranteeing zero violations under normal conditions.

动作掩码的意义：与其让智能体选一个错误的 ECU 然后给予惩罚，P4 直接把不合法的 ECU 从选项中"隐藏"掉。MaskablePPO 在采样前把被屏蔽动作的 logit 设为负无穷，保证在正常情况下零违约。

3a. _init_conflict_sets(K=10)

def _init_conflict_sets(self, K: int = 10) -> list[set]:
    sets = []
    for _ in range(K):
        j = random.randint(2, self.M)
        sets.append(set(random.sample(range(self.M), j)))
    return sets

Generates K=10 random "mutual-exclusion subsets". Each subset contains a random number (2 to M) of service indices. The rule: at most one service from each subset may land on the same ECU.

Analogy: Think of 10 rules like "services A, C, and F cannot share an ECU." This method randomly generates 10 such rules.

3b. _has_conflict(ecu_idx, svc_idx)

def _has_conflict(self, ecu_idx: int, svc_idx: int) -> bool:
    return svc_idx not in self.ecu_allowed[ecu_idx]

Checks whether placing service svc_idx onto ECU ecu_idx would create a conflict.

ecu_allowed[ecu_idx] is the current set of services still safe to place there. If svc_idx is not in that set, there's a conflict — return True.

3c. _update_ecu_allowed(ecu_idx, svc_idx)

def _update_ecu_allowed(self, ecu_idx: int, svc_idx: int) -> None:
    for subset in self.conflict_sets:
        if svc_idx in subset:
            self.ecu_allowed[ecu_idx] -= (subset - {svc_idx})

After successfully placing service svc_idx on ECU ecu_idx, this method propagates the conflict constraint.

It loops through all conflict subsets. If svc_idx belongs to a subset, all other services in that subset (subset - {svc_idx}) are removed from ecu_allowed[ecu_idx] using set-difference (-=).

Example: subset = {A, C, F}. After placing A, both C and F are removed from this ECU's allowed list.

Why this is efficient 为什么高效： Rather than scanning all conflict rules at query time (_has_conflict), P4 precomputes and maintains the "allowed" set so that the check becomes a single O(1) set-membership test. Updates happen only when a placement is made.

P4 预先维护"可用集合"，让查询时只需 O(1) 的集合成员检查。更新只在放置时发生，而非每次查询。

4Episode Reset

def reset(self, seed=None, options=None):
    super().reset(seed=seed)
    if self._scenarios is not None:
        caps, reqs, _cs = random.choice(self._scenarios)
        self.ecus     = [ECU(f"ECU{i}", cap) for i, cap in enumerate(caps)]
        self.services = [SVC(f"SVC{i}", req) for i, req in enumerate(reqs)]
        self.initial_vms = np.array([e.capacity for e in self.ecus], dtype=np.float32)
        self.conflict_sets = [set(cs) for cs in _cs]
    else:
        self.conflict_sets = self._init_conflict_sets()

    # Sort services descending by requirement (FFD order: hardest first).
    # Remap conflict_sets indices to match new order so conflict tracking stays
    # consistent with self._step (which now indexes sorted services).
    sort_idx = sorted(range(self.M), key=lambda i: -self.services[i].requirement)
    self.services = [self.services[i] for i in sort_idx]
    inv_perm = [0] * self.M
    for new_i, old_i in enumerate(sort_idx):
        inv_perm[old_i] = new_i
    self.conflict_sets = [{inv_perm[k] for k in cs} for cs in self.conflict_sets]

    self.remaining_vms   = self.initial_vms.copy()
    self.ecu_placements  = [set() for _ in range(self.N)]
    self.ecu_allowed     = [set(range(self.M)) for _ in range(self.N)]
    self.ar              = 0.0
    self._total_ru       = 0.0
    self._step           = 0
    self.capacity_violations = 0
    self.conflict_violations = 0
    return self._obs(), {}

Random scenario sampling: During training, each episode may use a different (capacity, demand) combination from a pre-built scenario library. This prevents the agent from memorizing one layout and forces it to generalize.

Sort descending by requirement: Greedy scheduling heuristic — tackle the largest, hardest-to-place services first, reducing infeasibility later in the episode.

Index remapping: After sorting, service indices change. sort_idx[i] = old_i means "new position i came from original index old_i." inv_perm[old_i] = i is the inverse — it maps original indices to new ones, used to remap each conflict set so rules remain correct after reordering.

ecu_allowed init: Each ECU's allowed set is reset to set(range(M)) — all services are initially placeable on all ECUs since no placements have been made yet.

def action_masks(self) -> np.ndarray:
    if self._step >= self.M:
        return np.zeros(self.N, dtype=bool)
    svc = self.services[self._step]
    mask = np.array(
        [(self.remaining_vms[j] >= svc.requirement) and (not self._has_conflict(j, self._step))
         for j in range(self.N)],
        dtype=bool,
    )
    # If no valid ECU exists, returns all-False; MaskablePPO handles this edge case.
    return mask
  1. remaining_vms[j] >= svc.requirement
  2. not _has_conflict(j, step)

Main mask logic: For each ECU j, mask[j]=True if and only if BOTH conditions hold:

  1. remaining_vms[j] >= svc.requirement: Enough remaining capacity (capacity constraint)
  2. not _has_conflict(j, step): No conflict exists (conflict constraint)

All-False edge case: If no ECU satisfies both conditions (very rare with FFD ordering), the mask returns all-False. MaskablePPO has internal safeguards against empty masks. The descending FFD service sort greatly reduces the likelihood of this scenario.

How MaskablePPO uses this: After the neural network outputs logits for all N actions, logits for False-masked actions are set to −∞. After softmax, their probability becomes 0 — the agent can never pick them.

P4 是强制零违约 · P4 guarantees zero violations:
由于动作掩码覆盖了容量和冲突两个约束，正确训练后的 P4 智能体在推断时几乎不会产生任何违约。FFD 排序进一步降低了遇到空掩码边缘情况的概率。

Because action masking covers both capacity and conflict constraints, a correctly trained P4 agent produces virtually zero violations at inference time. FFD ordering further reduces the probability of encountering the all-False mask edge case.

def _obs(self) -> np.ndarray:
    max_cap   = float(np.max(self.initial_vms) + 1e-8)
    total_cap = float(np.sum(self.initial_vms) + 1e-8)
    # ... (if episode done, set scalars to 0) ...

    svc = self.services[self._step]
    service_demand_norm = svc.requirement / max_cap           # scalar [0]
    conflict_flag = [float(self._has_conflict(j, self._step))  # [6+2N:6+3N]
                     for j in range(self.N)]
    valid_flag = self.action_masks().astype(np.float32)        # [6+4N:6+5N]

    remaining_abs_norm = np.clip(self.remaining_vms, -max_cap, max_cap) / max_cap
    initial_cap_pct    = self.initial_vms / max_cap
    ecu_allowed_frac   = [len(self.ecu_allowed[j]) / self.M for j in range(self.N)]

    return np.concatenate([
        [service_demand_norm],          # [0]
        [self.ar],                        # [1]
        [remaining_usable_capacity_sum], # [2]
        [remaining_service_demand_sum],  # [3]
        [remaining_usable_ecu_count],    # [4]
        [remaining_services_count],      # [5]
        initial_cap_pct,                 # [6 : 6+N]
        remaining_abs_norm,             # [6+N : 6+2N]
        conflict_flag,                   # [6+2N : 6+3N]
        ecu_allowed_frac,               # [6+3N : 6+4N]
        valid_flag,                      # [6+4N : 6+5N]
        remaining_svcs,                  # [6+5N : 6+5N+M]
    ]).astype(np.float32)

np.clip(..., -max_cap, max_cap)：限制剩余容量的范围。当 remaining_vms 为负数（服务超额放置，fallback 场景）时，clip 防止异常大的负值。

The observation vector is what the agent "sees" at each step. All values are normalized to roughly [-1, 1] — crucial for neural network training to avoid exploding gradients.

1e-8: Small epsilon added to denominators to prevent division-by-zero.

np.clip(..., -max_cap, max_cap): Clamps remaining capacity. In fallback scenarios where a service is placed on an over-capacity ECU, remaining_vms can go negative; clip prevents huge outlier values.

7Step & Reward

def step(self, action: int):
    svc = self.services[self._step]
    ar_prev = self.ar  # capture before placement for Δar reward

    cap_violated      = bool(self.remaining_vms[action] < svc.requirement)
    conflict_violated = self._has_conflict(action, self._step)

    # With action masking, violations should not occur during normal training.
    # ru is still computed for AR tracking; violations set ru=0 as safety net.
    ru = 0.0 if (cap_violated or conflict_violated) else \
        svc.requirement / (self.initial_vms[action] + 1e-8)

    self.remaining_vms[action] -= svc.requirement
    _was_empty = not self.ecu_placements[action]
    self.ecu_placements[action].add(self._step)
    if _was_empty:
        self._n_active += 1
    self._update_ecu_allowed(action, self._step)
    if ru > 0:
        self._total_ru += ru
    _active = self._n_active
    self.ar = self._total_ru / _active if _active > 0 else 0.0
    self._step += 1

    done = self._step >= self.M
    total_viol = self.capacity_violations + self.conflict_violations

    # Potential-based shaped reward: Δar gives immediate per-step credit.
    # Terminal bonus = ar_final for clean episodes → episode return = 2 * ar_final.
    delta_ar = self.ar - ar_prev
    terminal_bonus = self.ar if (done and total_viol == 0) else 0.0

    return self._obs(), float(delta_ar + terminal_bonus), done, False, info

terminal_bonus：只在最后一步（done=True）且零违约时才加 +ar_final。有违约时 bonus=0（不再是 +0.1*ar）。加上每步的 Δar 求和 = ar_final，整集回报 = 2 × ar_final，形成稳定的 value function bootstrap。

Δar (potential-based shaping): = ar_new − ar_prev. The per-step reward is the incremental AR change, not raw ru. This is a standard potential-based reward shaping trick: it provides dense gradient signal while provably preserving the optimal policy (potential function = current AR).

ru (resource utilization): Still computed internally for AR tracking. Set to 0 if a violation occurred — violations are excluded from the AR accumulator, keeping AR a pure measure of compliant placements.

AR (average resource utilization): = sum of valid ru / number of active ECUs (active = has at least one placed service). Primary performance metric.

terminal_bonus: Added only at the last step when total violations = 0. Sum of all Δar over the episode equals ar_final, so total episode return = 2 × ar_final — a stable bootstrap target for the value function. If any violation occurred, bonus = 0 (not the old +0.1*ar).

def _make_p4_env(seed: int) -> Monitor:
    """Module-level factory."""
    import random
    random.seed(seed)

Why a factory function: DummyVecEnv accepts a list of callables (not env instances) so it can construct environments internally. The factory is a recipe; DummyVecEnv calls it N times.

ActionMasker wrapper: Wraps P4Env into a Gymnasium env that exposes action masks. The _mask_fn parameter tells the wrapper how to retrieve the mask (call env.action_masks()). MaskablePPO requires this wrapper to function.

Monitor wrapper: Standard SB3 wrapper that automatically logs episode rewards, lengths, etc., making them accessible to training callbacks.

Different seeds: Each of the N parallel envs gets its own seed (C.SEED + i) to ensure diverse random behavior across environments.

def train_maskppo(ecus, services, device: str):
    n_envs = max(1, int(C.N_ENVS))
    env = DummyVecEnv(
        [functools.partial(_make_p4_env, C.SEED + i) for i in range(n_envs)],
    )
    cb  = P4Callback()
    model = MaskablePPO(
        policy        = "MlpPolicy",
        env           = env,
        learning_rate = C.PPO_LR,
        n_steps       = C.PPO_N_STEPS,
        batch_size    = C.PPO_BATCH_SIZE,
        n_epochs      = C.PPO_N_EPOCHS,
        gamma         = C.PPO_GAMMA,
        gae_lambda    = C.PPO_GAE_LAMBDA,
        clip_range    = C.PPO_CLIP_RANGE,
        ent_coef      = C.PPO_ENT_COEF,
        policy_kwargs = dict(net_arch=C.PPO_NET_ARCH),
        device        = device,
        verbose       = 0,
        seed          = C.SEED,
    )
    model.learn(total_timesteps=C.TOTAL_STEPS, callback=cb)
