1 What is DQN?

Core Concept

DQN is a reinforcement learning algorithm. At each step the agent picks an action, the environment gives a reward, and the agent's goal is to maximise total long-term reward.

In this project:

  • Action
  • State / Observation
  • Reward
  • 与 PPO/P4 的最大区别：DQN 没有动作屏蔽（Action Masking），违规动作仍然被执行，靠惩罚分数让 Q 网络自己学会避开非法动作

ParameterValueMeaning
DQN_LR 1e-3 Learning rate: step size for each network update. Too high = unstable; too low = slow.
DQN_BUFFER_SIZE 100,000 Replay buffer capacity: stores up to 100 K transitions (s, a, r, s') for random sampling.
DQN_LEARNING_STARTS 2,000 Warm-up: first 2 000 steps are random, filling the buffer before training begins.
DQN_BATCH_SIZE 64 Mini-batch size: 64 transitions sampled randomly from the buffer per update.
DQN_TAU 1.0 τ=1.0 = hard copy (full overwrite of target network); τ<1 would be soft/polyak update.
DQN_GAMMA 0.99 Discount factor: weight of future rewards. 0.99 = strongly future-oriented.
DQN_TRAIN_FREQ 4 Network update frequency: update once every 4 environment steps.
DQN_TARGET_UPDATE 500 Sync target network every 500 steps to stabilise training targets.
DQN_EXPLORATION_FRACTION 0.5 First 50% of training decays ε from 1.0 → 0.0 (exploration → exploitation).
DQN_EXPLORATION_FINAL_EPS 0.0 Final epsilon = 0: fully greedy at end of training.
DQN_NET_ARCH [128, 128] Network architecture: 2-hidden-layer MLP, each with 128 units.

3 Docstring & Design Philosophy

"""
DQN Environment — heavy penalty for violations + terminal bonus.

DQN (SB3) does not support action masking natively.  Constraint enforcement
is achieved via strong reward penalties.  The valid-action flag in the
observation lets the Q-network learn to avoid invalid actions.

Constraints:
    - Capacity violation → heavy penalty (-2.0), placement still proceeds
      (remaining_vms may go negative; visible in observation).
    - Conflict violation → heavy penalty (-2.0), placement still proceeds.
    - A well-trained agent should achieve zero violations on feasible scenarios.

Terminal bonus:
    +AR if zero violations, -AR if any violations.
"""
Key Design Decision

DQN does not support action masking. Violations still execute. The -2.0 penalty is the agent's "pain signal" — it must discover which actions are bad through trial and error.

def __init__(self, ecus: list[ECU], services: list[SVC], scenarios=None):
    super().__init__()
    self._scenarios = scenarios   # optional: training scenario list
    self.ecus     = ecus           # N ECU objects
    self.services = services       # M service objects
    self.N = len(ecus)
    self.M = len(services)

    # action space: choose one ECU from 0 to N-1
    self.action_space = gym.spaces.Discrete(self.N)

    # observation space: float vector of shape 5N+6+M, range [-1, 1]
    self.observation_space = gym.spaces.Box(
        low=-1.0, high=1.0, shape=(5 * self.N + 6 + self.M,), dtype=np.float32,
    )
Action Space vs Observation Space
  • Discrete(N)
  • Box(shape=5N+6+M)：观测向量有 5N+6+M 个浮点数，描述当前状态
  • 例：N=10, M=15 → 观测维度 = 5×10+6+15 = 71

6 Observation Vector

def _obs(self) -> np.ndarray:
    max_cap   = float(np.max(self.initial_vms) + 1e-8)   # avoid division by zero
    total_cap = float(np.sum(self.initial_vms) + 1e-8)

    # current service demand (normalised)
    svc = self.services[self._step]
    service_demand_norm = np.float32(svc.requirement / max_cap)

    # conflict flag: 1 = placing current service here violates a conflict constraint
    conflict_flag = np.array([float(self._has_conflict(j, self._step))
                               for j in range(self.N)], dtype=np.float32)

    # capacity-valid flag: 1 = ECU has sufficient remaining capacity
    valid_flag = (self.remaining_vms >= svc.requirement).astype(np.float32)

    return np.concatenate([
        [service_demand_norm],          # [0]       current service demand
        [self.ar],                      # [1]       current AR
        [remaining_usable_capacity_sum],# [2]       total usable remaining capacity
        [remaining_service_demand_sum], # [3]       total remaining service demand
        [remaining_usable_ecu_count],   # [4]       fraction of ECUs with enough capacity
        [remaining_services_count],     # [5]       fraction of services not yet placed
        initial_cap_pct,                # [6:6+N]   per-ECU initial capacity fraction
        remaining_abs_norm,             # [6+N:6+2N]  remaining capacity (may be negative!)
        conflict_flag,                  # [6+2N:6+3N] conflict flag
        ecu_allowed_frac,               # [6+3N:6+4N] fraction of conflict-free services per ECU
        valid_flag,                     # [6+4N:6+5N] capacity-valid flag
        remaining_svcs,                 # [6+5N:6+5N+M] remaining service demand list
    ]).astype(np.float32)

7 Reward Calculation

def step(self, action: int):
    svc = self.services[self._step]   # current service to place

    # ① detect violations
    cap_violated      = bool(self.remaining_vms[action] < svc.requirement)
    conflict_violated = self._has_conflict(action, self._step)

    # ② compute penalty (stackable! both violations = -4.0)
    cap_penalty      = -2.0 if cap_violated      else 0.0
    conflict_penalty = -2.0 if conflict_violated else 0.0

    # ③ utilisation reward: ru=0 on violation (no positive reward)
    ru = 0.0 if (cap_violated or conflict_violated) else \
         svc.requirement / (self.initial_vms[action] + 1e-8)

    # ④ execute placement regardless of violation (remaining_vms may go negative!)
    self.remaining_vms[action] -= svc.requirement
    self.ecu_placements[action].add(self._step)
    self._update_ecu_allowed(action, self._step)

    # ⑤ terminal bonus (triggered only on the last step)
    done = self._step >= self.M
    terminal_bonus = 0.0
    if done:
        terminal_bonus = self.ar if total_viol == 0 else -self.ar

    return self._obs(), float(ru + cap_penalty + conflict_penalty + terminal_bonus), ...

Valid Placement

  • cap_penalty = 0
  • conflict_penalty = 0
  • ru = req / cap_initial (positive)
  • remaining_vms decremented normally
  • Total step reward = +ru

Both Violated

  • cap_penalty = -2.0
  • conflict_penalty = -2.0
  • ru = 0.0
  • remaining_vms goes negative!
  • Total step reward = -4.0

8 Terminal Bonus

if done:
    # zero violations → extra reward +AR (encourages clean episodes)
    # violations present → extra penalty -AR (punishes failed episodes)
    terminal_bonus = self.ar if total_viol == 0 else -self.ar
？ Why Terminal Bonus?

Terminal bonus acts as an episode-level signal. +AR rewards zero-violation episodes heavily; -AR punishes any violation. This is harsher than P4 (which gave +0.1×AR even with violations).

9 Conflict Constraint Logic

def _init_conflict_sets(self, K: int = 10) -> list[set]:
    # randomly generate K conflict sets, each containing 2–M services
    # services in the same conflict set cannot share an ECU
    sets = []
    for _ in range(K):
        j = random.randint(2, self.M)
        sets.append(set(random.sample(range(self.M), j)))
    return sets

def _has_conflict(self, ecu_idx: int, svc_idx: int) -> bool:
    # returns True if placing svc_idx on ecu_idx violates a conflict constraint
    return svc_idx not in self.ecu_allowed[ecu_idx]

def _update_ecu_allowed(self, ecu_idx: int, svc_idx: int) -> None:
    # after placing svc_idx, remove conflicting services from ecu_allowed
    for subset in self.conflict_sets:
        if svc_idx in subset:
            self.ecu_allowed[ecu_idx] -= (subset - {svc_idx})
Example

冲突集 = {SVC0, SVC2, SVC5}。如果把 SVC0 放到 ECU3，则 ECU3 的 ecu_allowed 里会删除 SVC2 和 SVC5。之后试图把 SVC2 放到 ECU3 时，_has_conflict 返回 True，触发 -2.0 惩罚。

？ What is Experience Replay?

Why? Consecutive steps are highly correlated. Random sampling breaks this correlation, making training more stable — like shuffling your homework problems before reviewing.

Noneε-Greedy Exploration

DQN 使用 ε-贪心（epsilon-greedy） 策略来平衡探索（Exploration）与利用（Exploitation）：

探索 Explore (ε→1.0～0.0)　前 50% 步数
利用 Exploit (ε=0.0)　后 50%
阶段ε 值行为
Startε = 1.0完全随机选动作，大量探索环境
前 50% 训练1.0 → 0.0逐渐减少随机，增加依靠 Q 值决策
后 50% 训练ε = 0.0完全贪心：总选 Q 值最高的动作

if random() < epsilon:

else:

12 Environment Reset

def reset(self, seed=None, options=None):
    super().reset(seed=seed)
    if self._scenarios is not None:
        # randomly sample a training scenario
        caps, reqs, _cs = random.choice(self._scenarios)
        self.ecus     = [ECU(f"ECU{i}", cap) for i, cap in enumerate(caps)]
        self.services = [SVC(f"SVC{i}", req) for i, req in enumerate(reqs)]

    # sort services by demand descending (like First Fit Decreasing)
    self.services = sorted(self.services, key=lambda s: s.requirement, reverse=True)

    self.remaining_vms   = self.initial_vms.copy()   # reset remaining capacity
    self.ecu_placements  = [set() for _ in range(self.N)]  # clear placement records
    self.ecu_allowed     = [set(range(self.M)) for _ in range(self.N)]  # all services initially allowed
    self.ar              = 0.0
    self._step           = 0
    self.capacity_violations  = 0
    self.conflict_violations  = 0
    return self._obs(), {}
