1 DDQNEnv 文档说明 Docstring — The Environment is Identical

"""
DDQN Environment — heavy penalty for violations + terminal bonus.

N < M: each ECU hosts multiple services.

Identical constraint-enforcement strategy to DQNEnv.  The algorithm
distinction (dual Q-networks to reduce overestimation) is in the trainer,
not the environment.

Constraints:
    - Capacity violation → heavy penalty (-2.0), placement still proceeds.
    - Conflict violation → heavy penalty (-2.0), placement still proceeds.
    - A well-trained agent should achieve zero violations on feasible scenarios.

Terminal bonus (lt-specific adaptation, aligns reward scale with PPO methods):
    +AR if zero violations, -AR if any violations.
    Episode always runs M steps (no early termination).
"""
核心要点 Key Point

文档第一句就说了："Identical constraint-enforcement strategy to DQNEnv"。DDQNEnv 的环境逻辑与 DQNEnv 完全一样。DDQN 的算法改进（双网络减少高估）发生在训练器（Trainer）里，而不是环境里。

The docstring explicitly states the environment is identical to DQNEnv. The algorithm difference (Double Q-network update rule) lives entirely in the trainer, not the env.

3 为什么环境相同但结果不同？ Why Same Env, Different Results?

答案：Q 值更新公式不同！The Q-value update rule is different!

DQN 和 DDQN 使用完全相同的环境收集经验，但在更新 Q 网络时使用了不同的公式。这个差异导致 DDQN 的 Q 值更保守、更准确。

DQN 更新公式

DQN Target Calculation
target = r + γ × maxa Qtarget(s', a)

# 目标网络同时负责：
# 1. 选择最优动作（argmax）
# 2. 评估该动作的价值（Q value）
# 同一个网络做两件事 → 高估偏差！

DDQN 更新公式

DDQN Target Calculation
a* = argmaxa Qonline(s', a) ← 在线网络选动作
target = r + γ × Qtarget(s', a*) ← 目标网络评估价值

# 分离"选择"和"评估"两个职责
# 两个独立网络互相制衡 → 减少高估！

DQN — 单网络做两件事

目标网络 Q_target(s', a)
↓
max_a → 选最大值的动作
↓
同一网络评估该动作 Q 值
↓
容易选到噪声高估的动作并高估其价值

DDQN — 两网络分工

在线网络 Q_online(s', a)
↓
argmax → 选它认为最好的动作 a*
↓
目标网络评估 Q_target(s', a*)
↓
目标网络"纠正"在线网络的乐观估计

4 高估偏差的具体例子 Concrete Overestimation Example

场景：下一状态 s' 有两个可能动作 a1, a2。由于神经网络估计有噪声：

1
真实 Q 值（理想情况）： Q(s', a1) = 0.85，Q(s', a2) = 0.80
2
带噪声的网络估计： Q_hat(s', a1) = 0.90，Q_hat(s', a2) = 0.78（a1 被高估了！）
DQN
目标网络选 max → 选 a1（Q=0.90，被噪声抬高）
用同一网络评估 → 目标值 = 0.90 （高估！比真实 0.85 高）
结果：网络学到了虚假乐观的 Q 值
DDQN
在线网络选 argmax → 选 a1（在线网络认为 a1 最好）
目标网络评估 Q_target(s', a1) → 得到 0.85（目标网络估计更保守）
结果：目标值 = 0.85，更接近真实 Q 值
结论 Conclusion： DDQN 用在线网络"提名"最优动作，用目标网络"独立评分"，两者互相制衡，避免了"提名者即裁判"的问题。
DDQN separates "who picks the action" from "who evaluates it" — like having a separate judge who doesn't know who nominated the candidate.

5 环境逻辑完全一致 Identical Environment Logic

为了让读者清楚，下面展示 DDQNEnv 的 step() 方法——它与 DQNEnv 逐行相同：

def step(self, action: int):
    svc = self.services[self._step]

    # ① 违规检测（与 DQNEnv 完全相同）
    cap_violated      = bool(self.remaining_vms[action] < svc.requirement)
    conflict_violated = self._has_conflict(action, self._step)

    # ② 计算惩罚（与 DQNEnv 完全相同）
    cap_penalty      = -2.0 if cap_violated      else 0.0
    conflict_penalty = -2.0 if conflict_violated else 0.0
    ru = 0.0 if (cap_violated or conflict_violated) else \
         svc.requirement / (self.initial_vms[action] + 1e-8)

    # ③ 强制放置，remaining_vms 可能变负（与 DQNEnv 完全相同）
    self.remaining_vms[action] -= svc.requirement
    self.ecu_placements[action].add(self._step)
    self._update_ecu_allowed(action, self._step)

    # ④ 终局奖励（与 DQNEnv 完全相同）
    done = self._step >= self.M
    terminal_bonus = 0.0
    if done:
        terminal_bonus = self.ar if total_viol == 0 else -self.ar

    return self._obs(), float(ru + cap_penalty + conflict_penalty + terminal_bonus), ...

训练循环伪代码对比 Training Loop Pseudocode

DQN 训练循环

for batch in replay_buffer:
    s, a, r, s_next = batch

    # 目标网络同时选+评
    a_best = argmax_a Q_target(s_next)
    y = r + γ * Q_target(s_next, a_best)

    loss = (Q_online(s, a) - y)²
    optimizer.step(loss)

DDQN 训练循环

for batch in replay_buffer:
    s, a, r, s_next = batch

    # 在线选，目标评
    a_best = argmax_a Q_online(s_next)
    y = r + γ * Q_target(s_next, a_best)

    loss = (Q_online(s, a) - y)²
    optimizer.step(loss)

两者唯一的区别就是第 5 行：DQN 用 Q_target 选动作，DDQN 用 Q_online 选动作。这一行的改变导致了 Q 值高估程度的降低。

如何解读这个结果 How to Interpret

为什么 DDQN 违规更少？

  • 高估偏差少 → Q 网络不会"自信地选错" → 减少"confident but wrong"动作
  • 更保守的 Q 估计 → 面对模糊情况时决策更谨慎
  • 训练信号更稳定 → 网络收敛方向更准确

为什么 AR 差距不大？

  • 两者使用完全相同的环境和奖励函数
  • LT 场景约束极紧，两者都在探索接近极限的策略
  • AR 的改善需要从根本上解决无动作屏蔽的问题，而 DDQN 只优化了 Q 值精度
与 PPO+ActionMask 的比较 Comparison with PPO+ActionMask

无论是 DQN 还是 DDQN，在 LT 场景中的违规率都超过 60%，而使用动作屏蔽的 PPO（P4 方案）违规率接近 0%。这说明在约束严格的场景中，硬约束（动作屏蔽）远比软约束（惩罚）更有效。

In tight constraint scenarios, hard constraint enforcement (action masking) dominates soft constraint learning (reward penalties). DDQN's improvement over DQN is real but insufficient for constraint-critical applications.

9 直觉总结 Intuitive Summary

用一个类比来理解 DQN vs DDQN：

假设你要买股票，需要选出"最值得买"的一只并评估它的价格：

  • DQN：你自己选股票（觉得哪只涨幅会最高），然后自己给它定价。问题是你可能对自己看好的股票过于乐观（高估价格）。
  • DDQN：你选股票（在线网络推荐），然后让另一个独立分析师（目标网络）来评估这只股票的合理价格。独立分析师不会受你乐观情绪影响，定价更客观。

DQN: you pick and price the stock yourself → prone to overconfidence. DDQN: you pick, an independent analyst prices → more objective valuation.
