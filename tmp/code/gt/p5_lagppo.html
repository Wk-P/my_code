1Class Declaration & Core Idea

"""
P5 Environment — Hard capacity (masked), Lagrangian conflict (adaptive penalty).

Constraints:
    - Capacity violation → HARD (action masking; forced-overflow fallback
      triggers only when ALL ECUs are full, incurring a heavy penalty).
    - Conflict violation → Lagrangian (soft but adaptively penalised via λ;
      the training callback updates λ via dual ascent to drive violations → 0).
"""

class LagrangeEnv(gym.Env):
    """
    Observation (shape: 5N+7+M):
        ...
        [6+5N:6+5N+M]  remaining service demands (sorted descending)
        [6+5N+M]       current λ value, normalised by λ_max   ← NEW vs P4

    Reward per step:
        r_t = match_gain - (lambda_val + base_penalty) * c_t - forced_overflow_penalty
        terminal_bonus: +ar (zero violations) or -ar (some violations)   ← harsher than P4
    """

Lagrangian Relaxation is a classical operations research technique: move hard constraints into the objective function as penalty terms, converting a constrained optimization problem into a (more) unconstrained one.

In P5, conflict constraints aren't impenetrable walls — they're fines. The fine = λ per violation. As λ grows, the fine becomes heavier, and the agent increasingly prefers to avoid conflicts. By gradually raising λ, we force the agent to voluntarily comply with conflict constraints.

Analogy: P4 is like a locked gate (hard mask — can't pass). P5 is like a toll road that gets more expensive each time you take it (soft penalty that escalates via dual ascent).

def __init__(self, ecus: list[ECU], services: list[SVC],
             scenarios=None, lambda_init: float = 0.0,
             lambda_max: float = 10.0):
    super().__init__()
    # ... (same as P4) ...
    self.lambda_val = float(lambda_init)
    self.lambda_max = max(float(lambda_max), 1e-8)

    self.action_space = gym.spaces.Discrete(self.N)
    self.observation_space = gym.spaces.Box(

obs shape = 4N+7+M

max(..., 1e-8)

lambda_init=0.0: Training starts with λ=0, so conflict penalty is only base_penalty=0.2 (very small). The agent is free to explore and may initially choose conflicting actions.

lambda_max=10.0: Upper bound for λ, preventing penalty from growing without bound (which would cause gradient explosion). Normalization uses λ / λ_max to map λ into [0, 1].

obs shape = 4N+7+M: P5 removes the ecu_allowed_frac segment (present in P4's 5N+6+M), but adds λ_norm. Net change: −N (dropped ecu_allowed_frac) + 1 (added λ_norm) → 4N+7+M. This lets the neural network see the current penalty level and adapt its policy accordingly.

max(..., 1e-8): Prevents division-by-zero if someone accidentally passes lambda_max=0.

3λ · External Lambda Update

def set_lambda(self, val: float):
    self.lambda_val = float(val)

A deceptively simple method with elegant design. It allows a training callback (external to the env) to modify the environment's λ value.

Why not auto-update λ inside the env? With 40 parallel environments, λ should be updated based on the average violation rate across all environments. This statistic can only be computed after all envs have run for a period. The training callback collects stats from all envs, then calls set_lambda() on all of them uniformly.

Analogy: Like a teacher reviewing the whole class's test scores (all envs' violation rates) before deciding to raise the exam difficulty (λ value) for everyone next time.

4) · Conflict Helpers

def _init_conflict_sets(self, K: int = 10) -> list[set]:
    sets = []
    for _ in range(K):
        j = random.randint(2, self.M)
        sets.append(set(random.sample(range(self.M), j)))
    return sets

def _has_conflict(self, ecu_idx: int, svc_idx: int) -> bool:
    return svc_idx not in self.ecu_allowed[ecu_idx]

def _update_ecu_allowed(self, ecu_idx: int, svc_idx: int) -> None:
    for subset in self.conflict_sets:
        if svc_idx in subset:
            self.ecu_allowed[ecu_idx] -= (subset - {svc_idx})

These three conflict helper methods are identical to P4. The generation, checking, and updating of conflict sets works the same way.

The key difference is in how they're used: P4 uses _has_conflict() inside action_masks() to block actions. P5 uses _has_conflict() inside step() to determine whether to apply the Lagrangian penalty, but does not block the action.

In short: P5 still tracks conflicts (needs ecu_allowed), but delivers this information via "deduct points for violations" rather than "prevent the choice entirely."

5Reset and Lambda Persistence

def reset(self, seed=None, options=None):
    super().reset(seed=seed)
    if self._scenarios is not None:
        caps, reqs, _cs = random.choice(self._scenarios)
        self.ecus     = [ECU(f"ECU{i}", cap) for i, cap in enumerate(caps)]
        self.services = [SVC(f"SVC{i}", req) for i, req in enumerate(reqs)]
        self.initial_vms = np.array([e.capacity for e in self.ecus], dtype=np.float32)
        self.conflict_sets = [set(s) for s in _cs]
    else:
        self.conflict_sets = self._init_conflict_sets()

    # Sort services descending by requirement (FFD order).
    # Remap conflict_set indices so self._step always indexes sorted services.
    sort_idx = sorted(range(self.M), key=lambda i: -self.services[i].requirement)
    self.services = [self.services[i] for i in sort_idx]
    inv_perm = [0] * self.M
    for new_i, old_i in enumerate(sort_idx):
        inv_perm[old_i] = new_i
    self.conflict_sets = [{inv_perm[k] for k in cs} for cs in self.conflict_sets]

    self.remaining_vms   = self.initial_vms.copy()
    self.ecu_placements  = [set() for _ in range(self.N)]
    self.ecu_allowed     = [set(range(self.M)) for _ in range(self.N)]
    self.ar              = 0.0
    self._total_ru       = 0.0
    self._step           = 0
    self.episode_violations  = 0
    # NOTE: self.lambda_val is NOT reset here — persists across episodes!
    return self._obs(), {}

Other state variables (remaining_vms, ecu_placements, ecu_allowed, ar, _step, episode_violations) are reset normally for per-episode tracking.

λ is NOT reset! This is a critical design decision. At the start of each new episode, the λ updated by the callback persists. This ensures λ changes monotonically (only increasing, or decreasing slowly), never wiped back to zero by an episode reset.

If λ were reset to 0 each episode, the agent would always start each episode at "zero penalty" — dual ascent would be completely ineffective.

All other state variables (remaining_vms, ecu_placements, ecu_allowed, ar, _step, episode_violations) are reset normally to track per-episode statistics.

def action_masks(self) -> np.ndarray:
    if self._step >= self.M:
        return np.zeros(self.N, dtype=bool)
    svc = self.services[self._step]
    mask = (self.remaining_vms >= svc.requirement)   # ← capacity only!
    if not np.any(mask):
        best = int(np.argmax(self.remaining_vms))
        mask = np.zeros(self.N, dtype=bool)
        mask[best] = True
    return mask

Core difference between P4 and P5 masking:

P4: (remaining_vms[j] >= req) AND (not _has_conflict(j, step)) — checks both

P5: remaining_vms >= svc.requirement — checks capacity only

In P5, even if ECU j has a conflict, as long as it has sufficient capacity, mask[j]=True and the agent can select it. When it does, step() applies the Lagrangian penalty.

Why keep the capacity hard mask? Capacity is a hard physical constraint (a service truly cannot be placed on a full ECU). Conflict constraints in many real-world scenarios are softer (tradeable). P5's design philosophy: use guardrails for physical constraints, use economic incentives for soft constraints.

def _obs(self) -> np.ndarray:
    # ... (same as P4 for all segments [0] to [6+5N+M-1]) ...

    # valid_flag: capacity-only (no conflict check, unlike P4)
    valid_flag = (self.remaining_vms >= svc.requirement).astype(np.float32)

    # NEW vs P4: normalized lambda — tells agent how harsh the penalty is
    lambda_norm = np.float32(np.clip(self.lambda_val / self.lambda_max, 0.0, 1.0))

    ecu_allowed_frac = np.array(
        [len(self.ecu_allowed[j]) / self.M for j in range(self.N)],
        dtype=np.float32,
    )

    return np.concatenate([
        [service_demand_norm],          # [0]
        [self.ar],                        # [1]
        [remaining_usable_capacity_sum], # [2]
        [remaining_service_demand_sum],  # [3]
        [remaining_usable_ecu_count],    # [4]
        [remaining_services_count],      # [5]
        initial_cap_pct,                 # [6 : 6+N]
        remaining_abs_norm,             # [6+N : 6+2N]
        conflict_flag,                   # [6+2N : 6+3N]
        ecu_allowed_frac,               # [6+3N : 6+4N]
        valid_flag,                      # [6+4N : 6+5N]   ← capacity-only in P5
        remaining_svcs,                  # [6+5N : 6+5N+M]

    ]).astype(np.float32)

lambda_norm = clip(λ / λ_max, 0, 1)

valid_flag difference: P4's valid_flag = capacity AND no-conflict; P5's valid_flag = capacity only. P5 still provides conflict_flag to show which ECUs have conflicts, but it does not affect the mask.

lambda_norm = clip(λ / λ_max, 0, 1): Normalizes λ to [0, 1]. λ=0 → 0.0; λ=λ_max=10 → 1.0.

Why include λ in the observation? The agent's optimal policy depends on the current λ level:

  • Small λ: conflict penalty is light — the agent might rationally accept some conflicts for better match_gain
  • Large λ: conflict penalty is heavy — the agent should strictly avoid all conflicts

Without seeing λ, the agent cannot distinguish between "now I must avoid conflicts" and "conflicts are cheap right now" — severely hurting learning efficiency. By including λ_norm, the neural network can learn the meta-policy: "when λ_norm is high, be conservative."

valid_flag difference: P4's valid_flag = capacity AND no-conflict; P5's valid_flag = capacity only. P5 still provides conflict_flag to show which ECUs have conflicts, but this doesn't affect the mask.

8Lagrangian Reward Computation

def step(self, action: int):
    svc = self.services[self._step]
    ar_prev = self.ar  # capture before placement for Δar reward

    cap_violated      = bool(self.remaining_vms[action] < svc.requirement)
    conflict_violated = self._has_conflict(action, self._step)
    # λ (dual variable) tracks CONFLICT only; capacity gets a fixed large penalty.
    c_t = 1.0 if conflict_violated else 0.0

    ru = svc.requirement / (self.initial_vms[action] + 1e-8)
    self.remaining_vms[action] -= svc.requirement
    _was_empty = not self.ecu_placements[action]
    self.ecu_placements[action].add(self._step)
    if _was_empty:
        self._n_active += 1
    self._update_ecu_allowed(action, self._step)
    self._total_ru += ru  # always add (conflict penalized via λ, not ru=0)
    _active = self._n_active
    self.ar = self._total_ru / _active
    self._step += 1

    # Reward:
    #   Δar          — potential-based dense utilisation signal
    #   cap_penalty  — fixed -2.0 for capacity overflow
    #   Lagrangian   — adaptive conflict penalty via dual ascent
    #   terminal_bonus — +ar_final for clean episodes; 0 otherwise
    delta_ar     = self.ar - ar_prev
    cap_penalty  = -2.0 if cap_violated else 0.0
    base_penalty = 0.2
    terminal_bonus = self.ar if (done and self.episode_violations == 0) else 0.0
    reward = float(delta_ar + cap_penalty
                   - (self.lambda_val + base_penalty) * c_t
                   + terminal_bonus)
    return self._obs(), reward, done, False, {...}

P5 reward has four components:

1. Δar (potential-based shaping) = ar_new − ar_prev. Same approach as P4. ru is always accumulated into _total_ru (even on conflicts) so AR reflects actual resource occupation; conflict is penalized separately via λ rather than by zeroing ru.

Key difference from P4: P4 zeroes out ru on conflict. P5 always computes ru — this matches Lagrangian relaxation: original objective (AR) unchanged, penalty term added alongside.

2. lagrange_penalty = -(λ + 0.2) × c_t

  • c_t: conflict indicator — 1 if conflict, 0 otherwise
  • base_penalty = 0.2: ensures some penalty even when λ=0
  • lambda_val: adaptive component that grows via dual ascent

3. cap_penalty = -2.0 (capacity violations only). Capacity remains a hard constraint with a fixed large penalty.

4. terminal_bonus: Zero violations → +ar_final; any violations → 0.0 (not -ar). Combined with the Δar sum over the episode = ar_final, total episode return = 2 × ar_final — same bootstrap design as P4.

9Dual Ascent Explained

Training Loop:

  Episode 1..20 (λ=0.0)
  Agent explores freely, may produce many conflicts

        ↓ Every 20 episodes ↓

  P5Callback computes avg conflict rate
  ┌─────────────────────────────────────────────────────────┐
  │  avg_conflict_rate > 0%  →  λ ← λ + Δλ  (increase penalty)  │
  │  avg_conflict_rate = 0%  →  λ ← max(0, λ - ε)  (stabilise)  │
  └─────────────────────────────────────────────────────────┘
        ↓ Call ↓
  env.set_lambda(new_λ)  for each env in DummyVecEnv

  Episode 21..40 (λ=Δλ)
  Penalty slightly heavier, agent begins to weigh costs

        ↓ ... Repeat ...

  Episode N..N+20 (λ→λ_max)
  Penalty heavy enough, agent avoids conflicts voluntarily → violations → 0%

Dual Ascent is a classic convex optimization algorithm adapted here for reinforcement learning:

Mathematical intuition: In Lagrangian duality, λ is the "dual variable." The dual ascent direction equals the constraint violation magnitude — the more violations, the more λ should increase.

Convergence process:

  1. Small λ → conflicts cheap → agent ignores conflicts → high violation rate
  2. High violation rate → callback raises λ
  3. Large λ → conflicts expensive → agent avoids conflicts → low violation rate
  4. Zero violations → λ stabilizes (or decreases slightly)
  5. Repeat until converging to "zero violations, maximize AR" optimal policy

Advantage over P4: P5 allows the agent to "explore" violations during early training, potentially discovering better regions of policy space. The soft penalty also provides smoother gradient signals (P4's hard masking in fallback produces sparse, extreme gradients).

P5 terminal_bonus design: 0.0 vs P4's 0.0 (identical)



P5 terminal_bonus design: Same as P4 — 0.0 when any violation occurred. The Lagrangian penalty in each step already handles conflict discouragement; there's no need for an additional terminal punishment. Episode return = Σ Δar + ar_final (clean only) = 2 × ar_final, matching P4's bootstrap structure.
