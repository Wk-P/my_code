一 · Q 函数：每个选择"值多少分" — The Q-Function
中 Q 值的含义：
\(Q(s, a)\) = "在状态 s 下，做动作 a，然后按最优策略继续走，能拿到的总奖励期望"

举个例子：
假设现在是象棋残局（状态 s），你面前有两步可走（动作 a1, a2）：
  • \(Q(s, a1) = 100\)：走 a1 路线，预期总得分 100
  • \(Q(s, a2) = 60\)：走 a2 路线，预期总得分 60
显然选 a1。所以贪心策略就是：在每个状态下，选 Q 值最高的那个动作。

在 ECU 部署里：Q(s, j) = "当前状态下，把服务放到 ECU j，之后最优放置，总奖励期望值"
EN Meaning of Q-value:
\(Q(s, a)\) = "in state s, taking action a, then following the optimal policy — expected total reward"

Example:
In a chess endgame (state s) with two possible moves (a1, a2):
  • \(Q(s, a1) = 100\): path via a1 → expected total score 100
  • \(Q(s, a2) = 60\): path via a2 → expected total score 60
Choose a1. The greedy policy: in each state, pick the action with the highest Q-value.

In ECU deployment: Q(s, j) = "given current state, placing the service on ECU j, then playing optimally — expected total reward"
二 · 贝尔曼方程：从未来推现在 — The Bellman Equation
中 Q 值的递推关系（贝尔曼最优方程）：
\[ Q^*(s, a) = \mathbb{E}\!\left[ r + \gamma \cdot \max_{a'} Q^*(s', a') \;\Big|\; s, a \right] \]
逐项读：
  • \(Q^*(s, a)\)：在 s 下做 a 的最优 Q 值
  • \(r\)：立即得到的奖励
  • \(\gamma \cdot \max_{a'} Q^*(s', a')\)：到达下一个状态 s'，选最优动作 a'，对应的折扣 Q 值
直觉："现在的价值 = 立即收益 + 下一步最优价值"
就像：今天的总资产 = 今天的收入 + （折扣后的）未来最大资产。

用这个方程，AI 可以在不知道未来的情况下，通过"一步一步往后看"来估算每个状态-动作对的价值。
EN Recursive Q-value relationship (Bellman Optimality Equation):
\[ Q^*(s, a) = \mathbb{E}\!\left[ r + \gamma \cdot \max_{a'} Q^*(s', a') \;\Big|\; s, a \right] \]
Term by term:
  • \(Q^*(s, a)\): optimal Q-value for taking action a in state s
  • \(r\): immediate reward
  • \(\gamma \cdot \max_{a'} Q^*(s', a')\): discounted optimal Q-value in the next state s'
Intuition: "current value = immediate gain + (discounted) best future value"
Like: today's total wealth = today's income + (discounted) maximum future wealth.

With this equation, the AI can estimate the value of each state-action pair by "bootstrapping" from future estimates, without needing to know the future in advance.
三 · 用神经网络近似 Q 函数 — Neural Network Q-Approximation
中 为什么不直接列表存 Q 值？
状态空间（所有可能的状态）是连续的、高维的——有无数种可能的"剩余容量组合、需求组合"。 根本不可能用一张表列出所有 Q 值。

解决方案：用神经网络来"记忆"Q 函数
Q_\theta(s, a) \approx Q^*(s, a)
神经网络参数 \(\theta\) 就是网络的权重。
网络结构（本项目 DQN）：
  • 输入：状态向量 s（维度 5N+6+M）
  • 两层全连接：256 → 256，激活函数 ReLU
  • 输出：N 个数，分别对应每台 ECU 的 Q 值
选动作时：取输出里最大的那个对应的 ECU。
EN Why not just use a lookup table for Q-values?
The state space (all possible states) is continuous and high-dimensional — infinite combinations of remaining capacities and demands. A table is completely impractical.

Solution: use a neural network to "memorize" the Q-function
Q_\theta(s, a) \approx Q^*(s, a)
Network parameters \(\theta\) are the weights we train.
Network structure (this project's DQN):
  • Input: state vector s (dimension 5N+6+M)
  • Two fully connected layers: 256 → 256, ReLU activation
  • Output: N values, one Q-value per ECU
To select an action: pick the ECU with the highest output Q-value.
中 DQN 的学习目标（TD 目标）：
\[ y_t = r_t + \gamma \cdot \max_{a'} Q_{\theta^-}(s_{t+1}, a') \]
然后最小化预测误差：
\[ \mathcal{L}(\theta) = \mathbb{E}\!\left[\left(Q_\theta(s_t, a_t) - y_t\right)^2\right] \]
含义：让在线网络 \(Q_\theta\) 对当前动作的预测尽量接近"即时奖励 + 下一步最优价值"。
注意：\(y_t\) 使用的是目标网络 \(Q_{\theta^-}\) 而不是在线网络本身——具体见第六节。
EN DQN learning target (TD target):
\[ y_t = r_t + \gamma \cdot \max_{a'} Q_{\theta^-}(s_{t+1}, a') \]
Then minimize prediction error:
\[ \mathcal{L}(\theta) = \mathbb{E}\!\left[\left(Q_\theta(s_t, a_t) - y_t\right)^2\right] \]
Meaning: make the online network's \(Q_\theta\) prediction for the current action approach "immediate reward + best next-step value."
Note: \(y_t\) uses the target network \(Q_{\theta^-}\), not the online network — see Section 6.
四 · 探索与利用：ε-greedy 策略 — Exploration vs. Exploitation
中 困境：如果 AI 总是选 Q 值最高的动作，它永远学不到新东西。
类比：如果你只去过一家餐厅，你不知道别的餐厅是否更好；但如果你每次都随机选，永远找不到最好的那家。

ε-greedy 策略的折中方案：
\[ a_t = \begin{cases} \text{随机选一个 ECU} & \text{以概率 } \varepsilon \text{（探索）} \\ \arg\max_a Q_\theta(s_t, a) & \text{以概率 } 1-\varepsilon \text{（利用）} \end{cases} \]
ε 的退火（逐步降低）：
训练阶段ε 值行为
开始1.0完全随机探索，收集各种经历
中期逐步从1.0降到0.05越来越多地利用已学知识
末期0.0595% 贪心选最优，5% 随机探索
EN Dilemma: always picking the highest Q-value action means never learning anything new.
Analogy: if you only visit one restaurant, you'll never know if others are better; but if you always pick randomly, you'll never find the best one.

ε-greedy compromise:
\[ a_t = \begin{cases} \text{random ECU} & \text{with probability } \varepsilon \text{ (explore)} \\ \arg\max_a Q_\theta(s_t, a) & \text{with probability } 1-\varepsilon \text{ (exploit)} \end{cases} \]
ε annealing (gradual reduction):
Training phaseε valueBehavior
Start1.0Fully random exploration, diverse experience collection
MiddleDecreasing from 1.0 to 0.05Increasingly exploiting learned knowledge
Late0.0595% greedy, 5% random exploration
五 · 经验回放：打乱时序相关性 — Experience Replay
中 问题：序列数据的相关性会让训练不稳定
AI 按时序收集数据：s1→s2→s3→…，相邻状态高度相关（s2 几乎是 s1 的轻微变化）。 如果直接用这些序列数据训练，网络会"过拟合"到局部模式，训练不稳定。

解决方案：经验回放缓冲区（Replay Buffer）
1
创建一个大容量的"记忆库"（本项目 buffer 大小 = 10,000）
2
每步执行后，把 (s, a, r, s', done) 存入缓冲区
3
训练时，随机抽取 一个小批量（batch_size = 64）进行梯度更新
4
随机采样打破了时序相关性，相当于从过去所有经历中随机"回忆"
类比：不是只看昨天发生的事，而是随机翻开日记本的某一页——每次学习的场景都是独立的。
EN Problem: temporal correlations in sequential data destabilize training
AI collects data sequentially: s1→s2→s3→…, where adjacent states are highly correlated (s2 is almost s1 with tiny changes). Training directly on such sequences causes the network to overfit to local patterns, leading to instability.

Solution: Experience Replay Buffer
1
Create a large-capacity "memory" (buffer size = 10,000 in this project)
2
After each step, store (s, a, r, s', done) in the buffer
3
For training, randomly sample a mini-batch (batch_size = 64) for gradient updates
4
Random sampling breaks temporal correlation — equivalent to "randomly flipping to a page in your diary" from all past experiences
六 · 目标网络：稳定学习目标 — Target Network
中 问题：如果用在线网络生成学习目标，目标会随训练不断变动
想象你在追一个目标，但目标本身也在动——很难追上，容易来回震荡。
DQN 的目标 \(y_t = r + \gamma \max_{a'} Q_\theta(s', a')\) 如果 \(\theta\) 在持续更新，那 \(y_t\) 也在持续变化，学习很不稳定。

解决方案：目标网络 \(Q_{\theta^-}\)
  • 维护两个参数完全一样的网络：在线网络 \(Q_\theta\) 和目标网络 \(Q_{\theta^-}\)
  • 学习目标用目标网络生成：\(y_t = r + \gamma \max_{a'} Q_{\theta^-}(s', a')\)
  • 目标网络的参数每 500 步从在线网络复制一次（不是每步更新）
  • 在线网络每步更新，目标网络更新慢 → 目标相对固定 → 训练稳定
EN Problem: using the online network to generate targets creates a moving target
Imagine chasing a target that also keeps moving — hard to catch, prone to oscillation.
If \(\theta\) updates every step, then \(y_t = r + \gamma \max_{a'} Q_\theta(s', a')\) also changes every step → unstable learning.

Solution: Target Network \(Q_{\theta^-}\)
  • Maintain two identical-architecture networks: online \(Q_\theta\) and target \(Q_{\theta^-}\)
  • Generate targets from the target network: \(y_t = r + \gamma \max_{a'} Q_{\theta^-}(s', a')\)
  • Target network weights are copied from online every 500 steps (not every step)
  • Online updates frequently, target updates slowly → targets stay relatively fixed → stable training

    if random() < ε:
        a = random_ecu()
    else:
        a = argmax_j Q_θ(s, j)

    # 2. Execute action, get reward and next state
    s', r, done = env.step(a)

    # 3. Decay ε
    ε = max(ε_min, ε * 0.9995)

    # 4. If buffer large enough, sample and train

        loss = mean((Q_θ(s, a) - y)²)
        backprop, update θ

    # 5. Sync target network every 500 steps
    if step % 500 == 0:
        θ- = θ

    # 6. Reset if episode done
    if done:
        s = env.reset()
八 · 违规处理：终止惩罚机制 — Violation Handling
中 DQN 的约束处理方式（最简单、也最弱）： 
if 发生违规（容量超限 or 冲突）:
    r = -1.0    # 立即给一个负奖励
    done = True # 直接结束这个 episode
正常步骤的奖励：
r_t = \frac{\mathrm{req}_i}{\mathrm{cap}_j} \quad \text{（利用率增量）}
为什么这种方式在 LT 场景失败？
  • LT 场景里约束很紧，"正确路径"极窄。AI 需要精确地走完 15 步才能得到终局奖励。
  • 只要一步违规就全部清零（episode 结束），AI 获得完整奖励的机会极少。
  • 没有"还有多少约束空间"的指引，AI 基本靠撞墙来学习，效率极低。
  • 相比 P4（掩码后直接在合法空间探索），DQN 在 LT 下的违规率高达 68.1%。
EN DQN's constraint handling (simplest — and weakest): 
if violation occurs (capacity overflow or conflict):
    r = -1.0    # immediate negative reward
    done = True # terminate episode immediately
Normal step reward:
r_t = \frac{\mathrm{req}_i}{\mathrm{cap}_j} \quad \text{(utilization gain)}
Why does this fail in the LT scenario?
  • In LT, constraints are tight — the "correct path" is extremely narrow. DQN must complete all 15 placement steps to receive any meaningful reward.
  • One violation ends the episode immediately — very few complete episodes, very little useful signal.
  • No guidance on remaining constraint budget → AI learns almost entirely by hitting walls → very inefficient.
  • Compared to P4 (explores only in legal space), DQN's LT violation rate is 68.1%.
