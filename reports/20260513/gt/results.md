# 结果数据说明 — ecu_gt_svc_p（ECU > SVC，N=15, M=10）

## 1. 数据来源

`results.csv` 中的数据来自 `logs/multi_seed_results/aggregate_summary.csv`，为 5 个独立随机种子（seed=0,1,2,3,4）运行结果的均值和标准差。

每个 seed 对应一次完整的训练+评估流程：
- 训练集：160 个 scenario（80%）
- 测试集：40 个 scenario（20%），用于计算最终指标

---

## 2. 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `group` | str | 实验环境组名，本文件均为 `ecu_gt_svc_p` |
| `problem` | str | 问题编号 |
| `method` | str | 方法显示名称 |
| `n_seeds` | int | 参与统计的种子数量（均为 5） |
| `ar_mean_avg` | float | 跨 5 个 seed 的平均 AR（平均资源利用率） |
| `ar_mean_std` | float | AR 的跨 seed 标准差 |
| `placed_mean_avg` | float | 平均成功放置的服务数（最大值 M=10） |
| `placed_mean_std` | float | 放置数的跨 seed 标准差 |
| `viol_rate_avg` | float | 平均违反率（0=无违反，1=每步都违反） |
| `viol_rate_std` | float | 违反率标准差 |
| `cap_viol_avg` | float | 平均容量违反次数（40 个测试 episode 总计） |
| `cap_viol_std` | float | 容量违反标准差 |
| `conflict_viol_avg` | float | 平均冲突违反次数（40 个测试 episode 总计） |
| `conflict_viol_std` | float | 冲突违反标准差 |

**注意**：P6 的 `viol_rate_avg` 为空（NaN），因为其修复逻辑绕过了违反率统计逻辑。

---

## 3. 各方法数据解读

### ILP（最优基线）
- **AR = 0.6301 ± 0.0172**：gt 场景最优 AR，高于 eq（0.5418），因为 N=15 ECU 提供更多配对选项
- **标准差 = 0.0172**：高于 eq（0.0049），15 个 ECU 的场景多样性更大
- **违反率 = 0，全约束满足**

### P3 — 无约束 PPO
- **AR = 0.8735 ± 0.0200**：虚高，允许超载
- **冲突违反 = 111.6 次**（40 个 episode，平均每集 2.8 次）
- 容量违反 = 0（N=15 ECU 容量充裕，超载罕见）
- gt 场景违反总次数（111.6）少于 eq（143.4），因为 15 个 ECU 提供更多分散放置的机会，减少冲突

### P4 — 动作掩码 PPO
- **AR = 0.6155 ± 0.0160**，为 ILP 的 **97.7%**
- **零违反**
- placed_mean = 9.98（偶发 2/40 episode 有 1 服务无法放置）
- gt 场景中 P4 的 AR gap（2.3%）略大于 eq（1.3%），因为 N=15 使动作空间更大（Discrete(15)），策略搜索更难

### P5 — Lagrangian PPO
- **AR = 0.6169 ± 0.0166**，为 ILP 的 **97.9%**
- 冲突违反 = 1.6 次（接近零违反）
- 容量违反 = 0
- gt 场景中 P5 略优于 P4（AR 差 0.0014），且冲突违反仅 1.6 次，整体表现平衡

### P6 — PPO + 修复启发式
- **AR = 0.6178 ± 0.0172**，为 ILP 的 **98.0%**，三个可行 RL 方法中 AR 最高
- 容量违反 = 7.0 次（gt 场景中最高，远高于 eq 的 2.8 次）
- 冲突违反 = 1.0 次
- **容量违反增高原因**：N=15 时修复逻辑有更多备选 ECU，但部分小容量 ECU 被选为备选后触发超载

### DQN
- **AR = 0.6099 ± 0.0177**，为 ILP 的 **96.8%**
- 违反率 = 6.5%，容量违反 = 1.6，冲突违反 = 1.2
- 标准差最大（0.0177），稳定性最差

### DDQN
- **AR = 0.6100 ± 0.0171**，为 ILP 的 **96.8%**
- 违反率 = 4.5%，容量违反 = 1.0，冲突违反 = 0.8
- DQN 和 DDQN 在 gt 场景中几乎相同（AR 差 < 0.0001）

## 4. gt 场景综合排名

| 排名 | 方法 | AR | 违反率 | 综合评价 |
|------|------|-----|--------|----------|
| 1 | ILP | 0.6301 | 0% | 全局最优基准 |
| 2 | P6 Repair | 0.6178 | 0% (7次容量) | AR最高，有容量风险 |
| 3 | P5 Lagrange | 0.6169 | ~0% (1.6次) | 接近零违反，稳健 |
| 4 | P4 Mask | 0.6155 | 0% | 严格零违反 |
| 5 | DDQN | 0.6100 | 4.5% | 有违反，可接受 |
| 6 | DQN | 0.6099 | 6.5% | 略差于 DDQN |
| — | P3 | 0.8735 | 27.9% | 不可用 |

**gt 场景特点**：各可行方法（P4/P5/P6）的 AR 差异极小（0.6155~0.6178），均约为 ILP 的 97-98%。资源充裕时约束较易满足，方法间的分辨力降低。

## 5. 与 eq 场景的对比

| 指标 | eq | gt | 变化趋势 |
|------|----|----|----------|
| ILP 最优 AR | 0.5418 | 0.6301 | +16.3%（ECU增多，利用率提升） |
| P4 AR / ILP | 98.7% | 97.7% | 略下降（动作空间增大） |
| P5 AR / ILP | 99.1% | 97.9% | 略下降 |
| P3 冲突违反 | 143.4 | 111.6 | 减少（ECU充裕，分散放置减少冲突） |
| P5 冲突违反 | 0.6 | 1.6 | 略增（场景多样性增大，λ 收敛变难） |
| P6 容量违反 | 2.8 | 7.0 | 显著增加（修复逻辑在多 ECU 场景更激进） |
| ILP AR 标准差 | 0.0049 | 0.0172 | 增大（15 ECU 场景多样性更高） |

---

## 6. 数据可靠性说明

- **n_seeds = 5**：所有方法在 5 个独立种子下重复，统计结果可靠
- **标准差**：gt 场景的标准差普遍高于 eq（~0.015-0.020 vs ~0.004-0.007），因为 N=15 的场景多样性更大
- **测试集独立性**：测试 scenario 在训练中不可见，评估结果反映泛化能力
