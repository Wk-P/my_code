# 问题 4 — 带动作掩码的 PPO 服务部署

## 概述

问题 4 训练 **MaskablePPO**（带动作掩码扩展的 PPO）智能体，以**硬约束执行**方式解决服务部署任务。  
每步环境提供一个布尔掩码，将不可行的 ECU 分配标记为不可选。  
智能体在采样前，被屏蔽动作的策略 logits 被设为 $-\infty$，因此**约束违反在设计上不可能发生**。

---

## 问题定义

与问题 3 和 5 相同：

| 符号  | 含义                   |
| ----- | ---------------------- |
| $N$   | ECU 数量               |
| $M$   | 服务数量（$M \leq N$） |
| $e_j$ | ECU $j$ 的 VM 容量     |
| $n_i$ | 服务 $i$ 的 VM 需求    |

**目标：** $\max AR = \frac{1}{M}\sum_{i,j} x_{ij} \frac{n_i}{e_j}$

---

## MDP 建模

### 状态空间

$$s_t = \bigl[\, d_t,\; AR_t,\; r_0^t,\; \ldots,\; r_{N-1}^t \,\bigr] \in \mathbb{R}^{N+2}$$

| 索引           | 内容                  | 范围    |
| -------------- | --------------------- | ------- |
| 0              | 服务需求（归一化）    | $[0,1]$ |
| 1              | 累积 AR               | $[0,1]$ |
| $2 \ldots N+1$ | 各 ECU 剩余容量百分比 | $[0,1]$ |

### 动作掩码

在每步 `t`，`action_masks()` 返回长度为 $N$ 的布尔数组：

```
mask[j] = True   当且仅当  (ECU j 尚未使用)  且  (remaining_vms[j] >= n_t)
```

MaskablePPO 在 softmax 前应用掩码：

$$\pi(a_t \mid s_t) \propto \exp\!\bigl(\text{logit}(a_t)\bigr) \cdot \mathbf{1}[\text{mask}[a_t]]$$

### 奖励函数

$$r_t = \begin{cases} AR_M & t = M-1 \text{（最后一步）} \\ AR_{\text{当前}} & \text{无可行动作时（提前终止）} \\ 0 & \text{其他} \end{cases}$$

若某步没有任何 ECU 可行，回合提前终止，智能体获得当前已取得的 AR。

### 回合结构

```
reset()
  ├── 步骤 0：service_0 → ECU[a_0]（掩码过滤不可行 ECU）
  ├── 步骤 1：service_1 → ECU[a_1]
  ├── ...
  └── 步骤 M-1 或提前终止 → 奖励 = AR
```

---

## 约束保证

| 属性          | 保证                            |
| ------------- | ------------------------------- |
| 容量违反      | **不可能** — 掩码屏蔽超容量 ECU |
| 重复 ECU 分配 | **不可能** — 掩码屏蔽已分配 ECU |
| 违反率        | **0 %**（由构造保证）           |

---

## 算法与超参数

**算法：** MaskablePPO（`sb3-contrib`）

| 超参数        | 值           |
| ------------- | ------------ |
| 学习率        | `3e-4`       |
| `n_steps`     | `1024`       |
| 批次大小      | `128`        |
| 更新轮数      | `10`         |
| $\gamma$      | `0.999`      |
| GAE $\lambda$ | `0.95`       |
| 裁剪范围      | `0.2`        |
| Actor 网络    | `[128, 128]` |
| Critic 网络   | `[256, 256]` |
| 总训练步数    | `200,000`    |

Actor 和 Critic 使用**独立**网络结构（`dict(pi=[128,128], vf=[256,256])`）。

## 与其他方法的对比

|            | P2（ILP）    | P3（PPO，无约束） | **P4（PPO + 掩码）** | P5（PPO + 拉格朗日） |
| ---------- | ------------ | ----------------- | -------------------- | -------------------- |
| 约束执行   | 硬约束（LP） | 无                | **硬约束（掩码）**   | 软约束（惩罚）       |
| 违反率     | 0 %          | > 0 %             | **0 %**              | ≈ 0 %                |
| AR 性能    | 最优上界     | 高（但可能违约）  | **略低于 P3**        | 略低于 P3            |
| 不可行处理 | 不适用       | 分配照常进行      | 回合提前终止         | 分配照常进行         |

## 设计说明

- **动作掩码是强化学习方法中最简洁的约束处理策略**：无需奖励工程，且可证明实现零违反。
- 代价是：当无可行 ECU 时提前终止的回合会降低平均 AR，相比无约束的 P3 略低。
- 容量约束较紧的场景受益最大，因为随机选择会把大量步骤浪费在不可行 ECU 上。
